# 需求规格说明书 - 家属登录功能

## 1. 需求概述

### 1.1 需求背景
当前系统已实现用户登录和管理员登录功能，但家属（family_member表中的记录）无法独立登录系统。家属需要通过关联的用户账号才能查看和管理相关信息，这限制了家属的独立操作能力。

### 1.2 需求目标
实现家属独立登录功能，允许家属使用自己的手机号和密码登录系统，登录后可以：
- 查看和管理自己的健康信息
- 查看关联用户（如父母、子女）的健康状况
- 接收健康提醒和预警通知
- 作为紧急联系人接收紧急通知

### 1.3 需求范围
**包含内容：**
- 家属登录页面开发
- 家属认证系统实现
- 家属权限体系设计
- 数据库结构调整
- 后端接口开发

**不包含内容：**
- 家属注册功能（由用户添加家属）
- 家属信息修改功能（已存在）
- 第三方登录集成

## 2. 功能需求

### 2.1 家属登录功能
**功能描述：** 家属使用手机号和密码登录系统

**输入：**
- 手机号（必填，11位数字）
- 密码（必填，6-20位字符）

**处理流程：**
1. 验证手机号格式
2. 查询family_member表，验证手机号是否存在
3. 验证密码是否正确（BCrypt加密验证）
4. 生成家属专属JWT Token
5. 返回登录成功信息和Token

**输出：**
- 登录成功：返回Token、家属信息、关联用户信息
- 登录失败：返回错误提示信息

**业务规则：**
- BR-001: 手机号必须在family_member表中存在
- BR-002: 密码必须使用BCrypt加密存储
- BR-003: Token有效期24小时
- BR-004: 登录失败3次后锁定账号30分钟

### 2.2 家属权限管理
**功能描述：** 定义家属的角色权限

**权限级别：**
- 查看自己的健康记录
- 查看关联用户的健康记录（需用户授权）
- 接收健康提醒和预警
- 作为紧急联系人接收紧急通知

**权限限制：**
- 不能修改关联用户的健康记录
- 不能添加或删除其他家属
- 不能访问管理功能

### 2.3 家属信息展示
**功能描述：** 登录后展示家属的个人信息和关联用户信息

**展示内容：**
- 家属基本信息（姓名、年龄、性别、健康状况）
- 关联用户信息（姓名、关系、健康状况）
- 紧急联系人标识
- 最近健康记录

## 3. 非功能需求

### 3.1 安全性需求
- 密码必须BCrypt加密存储
- Token必须包含家属ID和手机号
- 所有接口必须验证Token有效性
- 敏感操作需要二次验证

### 3.2 性能需求
- 登录响应时间 < 2秒
- Token验证时间 < 100ms
- 支持并发登录请求

### 3.3 可用性需求
- 登录界面简洁明了
- 错误提示清晰准确
- 支持记住密码功能
- 支持自动登录功能

## 4. 数据需求

### 4.1 数据库变更需求

**需要修改的表：**

#### family_member表
需要添加以下字段：
- `password` VARCHAR(255) - 登录密码（BCrypt加密）
- `username` VARCHAR(50) - 登录用户名（可选）
- `login_enabled` TINYINT(1) DEFAULT 0 - 是否允许登录（0-否，1-是）
- `login_fail_count` INT DEFAULT 0 - 登录失败次数
- `lock_until` DATETIME NULL - 账号锁定截止时间
- `last_login_time` DATETIME NULL - 最后登录时间
- `token` VARCHAR(500) NULL - 当前Token（可选）

**需要新增的表：**

#### family_auth_log表（家属认证日志）
- `id` BIGINT PRIMARY KEY AUTO_INCREMENT
- `family_id` BIGINT - 家属ID
- `login_time` DATETIME - 登录时间
- `login_ip` VARCHAR(50) - 登录IP
- `login_device` VARCHAR(100) - 登录设备
- `login_result` TINYINT - 登录结果（1-成功，0-失败）
- `fail_reason` VARCHAR(200) - 失败原因

### 4.2 数据迁移需求
- 为现有家属生成默认密码（手机号后6位）
- 设置login_enabled为0，需要用户主动开启
- 记录数据迁移日志

## 5. 接口需求

### 5.1 家属登录接口
**接口路径：** POST /api/family/login
**请求参数：**
```json
{
  "phone": "13800138000",
  "password": "123456"
}
```
**响应数据：**
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "familyInfo": {
      "id": 1,
      "name": "王秀英",
      "phone": "13800138000",
      "relation": "配偶",
      "isEmergencyContact": true
    },
    "relatedUser": {
      "id": 1,
      "name": "王建国",
      "healthCondition": "良好"
    }
  }
}
```

### 5.2 家属信息查询接口
**接口路径：** GET /api/family/info
**请求头：** Authorization: Bearer {token}
**响应数据：** 家属详细信息

### 5.3 家属健康记录查询接口
**接口路径：** GET /api/family/health-records
**请求头：** Authorization: Bearer {token}
**响应数据：** 家属的健康记录列表

## 6. 约束条件

### 6.1 技术约束
- 必须使用现有的JWT认证框架
- 必须兼容现有的权限管理系统
- 必须使用现有的数据库连接池
- 必须遵循现有的代码规范

### 6.2 业务约束
- 家属登录功能默认关闭，需用户主动开启
- 家属只能查看，不能修改关联用户的信息
- 紧急联系人自动获得登录权限
- 家属数量不受限制

### 6.3 时间约束
- 数据库变更需要提前备份
- 需要通知所有用户家属登录功能上线
- 需要提供家属登录使用说明

## 7. 验收标准

### 7.1 功能验收
- [ ] 家属可以使用手机号和密码成功登录
- [ ] 登录后可以查看自己的信息
- [ ] 登录后可以查看关联用户的信息
- [ ] Token验证机制正常工作
- [ ] 权限控制正确执行

### 7.2 安全验收
- [ ] 密码已BCrypt加密存储
- [ ] Token无法被伪造
- [ ] 登录失败3次后账号被锁定
- [ ] 敏感信息已脱敏处理

### 7.3 性能验收
- [ ] 登录响应时间 < 2秒
- [ ] Token验证时间 < 100ms
- [ ] 支持100并发登录请求

## 8. 风险分析

### 8.1 技术风险
- **风险：** 数据库迁移可能导致数据丢失
- **应对：** 提前备份数据库，分批次迁移

### 8.2 安全风险
- **风险：** 家属账号可能被暴力破解
- **应对：** 实现登录失败锁定机制，增加验证码

### 8.3 业务风险
- **风险：** 用户可能不愿意开启家属登录功能
- **应对：** 提供详细的使用说明和安全保障说明

## 9. 依赖关系

### 9.1 前置依赖
- 现有的用户登录系统
- 现有的JWT认证框架
- 现有的权限管理系统
- family_member表已存在

### 9.2 后置依赖
- 家属健康记录管理功能
- 家属消息通知功能
- 家属紧急联系人功能

## 10. 术语定义

- **家属（Family Member）：** 与用户有亲属关系的人员，记录在family_member表中
- **关联用户（Related User）：** 家属所关联的系统用户，通过user_id字段关联
- **紧急联系人（Emergency Contact）：** 标记为紧急联系人的家属，is_emergency_contact=1
- **JWT Token：** JSON Web Token，用于身份认证的令牌
- **BCrypt：** 一种密码加密算法，用于安全存储密码
