## 问题诊断

### 问题1：应用未重新编译

**症状**：修改代码后，应用行为没有变化

**检查方法**：
1. 在DevEco Studio中，查看 Build Output 窗口
2. 确认是否执行了 Rebuild Project
3. 查看编译时间是否是最近的

**解决方法**：
```
1. Build → Clean Project（等待完成）
2. Build → Rebuild Project（等待完成，可能需要1-2分钟）
3. 停止应用（Ctrl+F2）
4. 重新运行（Shift+F10）
```

### 问题2：网络配置问题

**症状**：后端没有收到任何请求

**检查方法**：
1. 在DevEco Studio的HiLog中搜索：`HttpUtil`
2. 查看实际请求的URL是什么
3. 查看是否有网络错误

**解决方法**：
- 如果URL还是 `http://10.0.2.2:8080`，说明配置没生效
- 需要再次Clean和Rebuild

### 问题3：Token缓存问题

**症状**：显示"登录已过期"

**检查方法**：
- 查看应用本地存储是否有旧Token

**解决方法**：
- 清除应用数据
- 或卸载应用重新安装

## 完整解决步骤

### 步骤1：确认文件已修改

```typescript
[EnvType.DEV]: { BASE_URL: 'http://10.0.9.202:8080', UPLOAD_URL: 'http://10.0.9.202:8080/upload' }
```

**Register.ets**（发送验证码部分）：
```typescript
const response = await HttpUtil.post<void>('/user/send-code', sendCodeRequest, { skipAuth: true });
```

### 步骤2：完全重新编译

在DevEco Studio中：

1. **停止应用**
   - 点击红色停止按钮
   - 或按 Ctrl+F2

2. **清理项目**
   - 菜单：Build → Clean Project
   - 等待清理完成（底部会显示进度）

3. **重新构建**
   - 菜单：Build → Rebuild Project
   - 等待构建完成（可能需要1-2分钟）
   - 查看Build Output，确保没有错误

4. **运行应用**
   - 点击绿色运行按钮
   - 或按 Shift+F10
   - 等待应用启动

### 步骤3：查看HiLog日志

应用启动后：

1. 在DevEco Studio底部，点击 **HiLog** 标签
2. 在搜索框输入：`HttpUtil`
3. 点击"获取验证码"按钮
4. 查看日志输出：
   - 应该看到：`[HttpUtil] POST http://10.0.9.202:8080/user/send-code`
   - 如果看到 `http://10.0.2.2:8080`，说明配置没生效

### 步骤4：查看后端日志

在后端控制台：

1. 点击"获取验证码"按钮
2. 查看后端日志是否出现新内容：
   - 应该看到：`[验证码] 发送验证码到手机: 13948999643, 验证码: 123456`
   - 如果没有新日志，说明请求没到达后端

### 步骤5：测试验证码

如果以上都正常：

1. 输入手机号：`13948999643`
2. 点击"获取验证码"
3. 应该看到"验证码已发送"提示
4. 验证码输入框自动填充：`123456`
