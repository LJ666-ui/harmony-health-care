# 网络连接问题诊断指南

## 问题现象

前端应用显示错误：**"Failed to connect to the server"**

## 问题原因分析

### 原因1：运行环境不匹配（最常见）

**前端配置**：`http://10.0.2.2:8080`
- 这个地址**仅适用于Android模拟器**
- `10.0.2.2` 是模拟器访问宿主机的特殊地址

**如果您使用的是真机测试**：
- 真机无法识别 `10.0.2.2`
- 需要改为电脑的局域网IP地址

### 原因2：防火墙阻止连接

Windows防火墙可能阻止了8080端口的入站连接。

### 原因3：不在同一网络

手机和电脑不在同一局域网，无法通信。

## 解决方案

### 方案A：使用模拟器测试（推荐）

1. 确保使用Android模拟器运行应用
2. API配置保持不变：`http://10.0.2.2:8080`
3. 后端服务在本机运行

### 方案B：使用真机测试

#### 步骤1：获取电脑IP地址

运行诊断工具：
```bash
双击 check-network.bat
```

或手动查看：
```bash
ipconfig
# 找到 "IPv4 地址" 那一行
# 例如：192.168.1.100
```

#### 步骤2：修改前端配置

编辑文件：`entry/src/main/ets/common/constants/ApiConstants.ets`

修改第16行：
```typescript
// 修改前
[EnvType.DEV]: { BASE_URL: 'http://10.0.2.2:8080', UPLOAD_URL: 'http://10.0.2.2:8080/upload' }

// 修改后（替换为你的IP）
[EnvType.DEV]: { BASE_URL: 'http://192.168.1.100:8080', UPLOAD_URL: 'http://192.168.1.100:8080/upload' }
```

#### 步骤3：配置防火墙

**方法1：关闭防火墙（测试用）**
```bash
# Windows设置 → 更新和安全 → Windows安全 → 防火墙和网络保护 → 关闭防火墙
```

**方法2：开放端口（推荐）**
```bash
# 以管理员身份运行PowerShell
netsh advfirewall firewall add rule name="HealthCare Backend" dir=in action=allow protocol=tcp localport=8080
```

**方法3：添加应用例外**
```bash
# 控制面板 → Windows Defender 防火墙 → 允许应用通过防火墙
# 添加 java.exe 的例外
```

#### 步骤4：确保同一网络

- 手机和电脑连接同一个Wi-Fi
- 或手机连接电脑的热点

#### 步骤5：验证连接

在手机浏览器中访问：
```
http://你的IP:8080/swagger-ui.html
```

如果能打开Swagger页面，说明连接成功。

## 快速诊断步骤

1. **运行诊断工具**
   ```bash
   双击 check-network.bat
   ```

2. **检查后端服务**
   - 访问：http://localhost:8080/swagger-ui.html
   - 如果打不开，说明后端未启动

3. **检查网络连接**
   - 确认手机和电脑在同一网络
   - 在手机浏览器测试：http://电脑IP:8080

4. **检查防火墙**
   - 临时关闭防火墙测试
   - 如果成功，说明是防火墙问题

## 常见错误

### 错误1：IP地址错误

❌ 错误：使用 `127.0.0.1` 或 `localhost`
✅ 正确：使用局域网IP，如 `192.168.1.100`

### 错误2：端口错误

❌ 错误：后端运行在8081，前端配置8080
✅ 正确：前后端端口一致

### 错误3：协议错误

❌ 错误：前端用 `https`，后端用 `http`
✅ 正确：协议一致，都用 `http`

## 测试验证

### 后端测试
```bash
curl http://localhost:8080/user/info?id=1
```

### 手机测试
在手机浏览器访问：
```
http://你的IP:8080/user/info?id=1
```

应该返回JSON数据。

## 完整检查清单

- [ ] 后端服务已启动（端口8080）
- [ ] 后端可本地访问（localhost:8080）
- [ ] 已获取电脑局域网IP
- [ ] 前端API地址已正确配置
- [ ] 手机和电脑在同一网络
- [ ] 防火墙已配置或关闭
- [ ] 手机浏览器可访问后端
