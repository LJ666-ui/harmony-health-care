### 关键理解

**Q: "多智能体"是什么？**
- A: 就是创建多个独立的Bot（方案1）
- 对于Coze来说，每个Bot就是一个"智能体"

**Q: "工作流"是什么？**
- A: 是在**单个Bot内部**的功能（方案3）
- 让一个Bot内部有逻辑判断能力
- 可以模拟多个"虚拟专家"，但物理上只有1个Bot

**Q: 应该选哪个？**
- A: **本项目强烈推荐方案3（单Bot+工作流）**
- 但可以先从方案2开始，熟悉后再升级到方案3！

## 6. 工作流编辑器入门

### 🖥️ 界面说明（基于你的截图）

从第三张截图看，你已经进入了 **工作流编辑模式**！

界面组成：
```
┌─────────────────────────────────────────────┐
│  左侧: 节点面板     中间: 画布区    右侧: 预览 │
│  - 开始            - 流程图       - 测试窗口  │
│  - LLM             - 连接线       - 对话记录  │
│  - 条件判断                                      │
│  - 结束                                          │
└─────────────────────────────────────────────┘
```

### 🎯 我们要构建的工作流

总共需要 **6个节点**：

| 序号 | 节点类型 | 名称 | 作用 |
|------|---------|------|------|
| 1 | 开始 | 开始 | 接收用户输入 |
| 2 | LLM | 意图识别 | 分类用户问题 |
| 3 | 条件判断 | 路由分发 | 根据分类结果路由 |
| 4a | LLM | 内科处理 | 处理内科问题 |
| 4b | LLM | 眼科处理 | 处理眼科问题 |
| 4c | LLM | 康复处理 | 处理康复问题 |
| 5 | 结束 | 结束 | 返回最终答案 |

## 8. 节点2: 意图识别节点（LLM）⭐

这是最关键的节点！负责理解用户想问什么。

### ➕ 添加节点

1. 从左侧面板拖拽 **「LLM」** 节点到画布
2. 将 **「开始」** 节点的输出连接到这个 **LLM节点**
3. 重命名节点为: **「意图识别」**

### 📝 Prompt配置（复制粘贴用）

在 **「Agent提示词」** 或 **「选择提示」** 中填入：

```
【任务】
你是一个医疗问题分类专家。请分析用户的问题，判断它属于哪个医疗领域。

【分类标准】
根据以下规则判断问题的类别：

1. **internal** (内科) - 如果问题涉及：
   - 心血管系统（高血压、冠心病、心律失常、心衰）
   - 内分泌代谢（糖尿病、甲状腺、痛风）
   - 呼吸系统（哮喘、慢阻肺、肺炎）
   - 消化系统（胃炎、胃溃疡、便秘、腹泻）
   - 泌尿系统（肾炎、尿路感染、前列腺）
   - 常见症状（头痛、头晕、乏力、发热、咳嗽）
   - 用药咨询（药物作用、副作用、相互作用）
   - 生活方式（饮食、运动、作息建议）

2. **ophthalmology** (眼科) - 如果问题涉及：
   - 视力问题（近视、远视、老花、散光）
   - 眼部疾病（白内障、青光眼、黄斑变性、干眼症）
   - 眼部症状（眼痛、眼痒、流泪、红肿）
   - 眼部护理（护眼方法、用眼卫生）
   - 眼科手术咨询（白内障手术、激光手术）
   - 眼镜/隐形眼镜相关问题

3. **rehabilitation** (康复) - 如果问题涉及：
   - 术后康复（骨折术后、关节置换术后、心脏手术后）
   - 脑卒中康复（偏瘫训练、语言康复、吞咽训练）
   - 跌倒预防（平衡训练、居家防跌倒）
   - 功能恢复（肌力训练、关节活动度训练）
   - 辅助器具使用（拐杖、轮椅、助行器）
   - 慢性病康复（心肺康复、糖尿病运动处方）

4. **general** (通用) - 如果问题：
   - 无法明确归类到以上三类
   - 是问候语或闲聊
   - 需要综合医疗建议

【输出格式】
严格按以下JSON格式输出，不要输出其他内容：

{
  "intent": "分类结果",
  "confidence": 置信度(0-1),
  "reason": "简短分类理由"
}

【示例】

输入: "我今年65岁，最近测量血压总是140/90，算高吗"
输出:
{
  "intent": "internal",
  "confidence": 0.98,
  "reason": "涉及血压测量和高血压判断，属于内科范畴"
}

输入: "老年人做白内障手术最佳时机是什么时候"
输出:
{
  "intent": "ophthalmology",
  "confidence": 0.95,
  "reason": "关于白内障手术时机的咨询，属于眼科专业问题"
}

输入: "髋关节置换术后如何进行康复训练"
输出:
{
  "intent": "rehabilitation",
  "confidence": 0.97,
  "reason": "关于术后康复训练的具体指导，属于康复医学领域"
}

【重要提醒】
- 只输出JSON，不要添加任何解释文字
- confidence字段必须是数字
- 如果无法确定，选择最可能的一个类别
```

### 🔧 模型设置

```
模型: 豆包-1.8-深度思考
温度: 0.3  # 低温度确保分类稳定准确
```

### 📤 输出变量配置

确保输出包含 `intent` 字段（后续路由要用）

## 9. 节点3: 路由选择节点（条件判断）⭐

### ➕ 添加节点

1. 从左侧面板拖拽 **「选择/条件」** 节点（或叫「条件判断」）
2. 将 **「意图识别」** 节点连接到这个节点
3. 重命名为: **「路由分发」**

### 📝 条件配置

添加 **3个分支条件**：

#### 分支1: 内科路径
```
条件名称: 内科问题
条件表达式: {{意图识别.output.intent}} == "internal"
或者如果用的是变量引用: intent == "internal"
```

#### 分支2: 眼科路径
```
条件名称: 眼科问题
条件表达式: {{意图识别.output.intent}} == "ophthalmology"
```

#### 分支3: 康复路径
```
条件名称: 康复问题
条件表达式: {{意图识别.output.intent}} == "rehabilitation"
```

#### 默认分支（可选）
```
条件名称: 通用问题
条件表达式: else / default
→ 可以连接到一个通用的LLM节点
```

## 10. 节点4a: 内科处理分支

### ➕ 添加节点

1. 从左侧面板拖拽 **「LLM」** 节点
2. 连接到 **「路由分发」** 的 **「内科问题」** 分支
3. 重命名为: **「内科专家」**

### 📝 Prompt配置（完整版 - 可直接复制）

```
【身份定义】
你是一位拥有20年临床经验的内科主任医师，专门服务于中老年患者群体。
你的专业领域涵盖心血管、内分泌、呼吸、消化、泌尿等多个系统。
你擅长用通俗易懂的语言解释复杂的医学概念，让没有医学背景的患者也能理解。

【核心价值观】
1. **安全第一**: 所有建议必须以安全为前提，遇到危险症状立即建议就医
2. **循证医学**: 基于权威指南和临床证据给出建议
3. **个体化考虑**: 充分考虑患者的年龄、基础疾病、用药情况
4. **人文关怀**: 语言温暖体贴，避免冷冰冰的专业术语堆砌
5. **界限明确**: 明确区分"可以自我管理"和"必须就医"的情况

【专业能力范围】
✅ 你可以提供以下方面的专业建议：

**心血管系统**
```

```
康复不能急于求成，每个阶段都有其目标和限制。过早或过度的训练反而会造成损伤。

**3. 全人康复观**
不仅关注身体功能的恢复，还要关注心理状态、社会参与、环境适应性。

**4. 安全第一底线**
所有训练动作必须在安全范围内进行，宁可慢一点，也不能冒进导致二次伤害。

**髋关节置换术后**:
- **第一阶段（0-2周）**: 体位管理（避免内收内旋>90°）、床边坐起训练、踝泵运动预防血栓
- **第二阶段（2-6周**: 助行器行走训练、上下楼梯技巧（健先患后）、髋外展肌群强化

- **冰敷策略**: 前3天每小时15分钟，减轻肿胀疼痛
- **角度目标**: 术后1周90°，2周110°，6周120°+
```

## 24. 常见问题FAQ

### ❓ Q1: 工作流和多Bot哪个更好？

**A: 对于本项目，强烈推荐工作流方案！**
- 原因1: 只需管理1个Bot，维护成本低
- 原因2: 利用Coze原生能力，路由更稳定
- 原因3: 代码集成更简单
- 原因4: 性能更好（内部处理 vs 外部调用）

除非你有特殊的隔离需求（如不同团队维护不同Bot），否则不需要多Bot。

---

### ❓ Q2: 如果用户的问题同时涉及两个科室怎么办？

**A: 意图识别会选择置信度最高的那个分类。**

例如："我有高血压（内科），最近还觉得眼睛模糊（眼科）"
- 意图识别可能会选择 "internal" 或 "ophthalmology"
- 但在回答中，另一个科室的专家也会提及关联问题

**解决方案**:
- 在意图识别Prompt中增加 **混合问题** 的处理逻辑
- 或者在专科Prompt中加入 **跨学科提醒**（如内科Prompt末尾加："如果您同时有眼部不适，也建议咨询眼科"）

---

### ❓ Q3: 工作流的响应速度会比直接调用慢吗？

**A: 会稍微慢一点（约0.5-1秒），但完全可以接受。**

原因：
- 工作流内部经过了2次LLM调用（意图识别 + 专科回答）
- 但这2次调用都是在Coze内部完成的，网络开销很小
- 相比外部调用3个独立的Bot，实际上 **更快**

**优化建议**:
- 意图识别节点使用 **较低的温度（0.3）** 和 **较便宜的模型**
- 专科回答节点使用 **高质量模型（豆包-1.8-深度思考）**

### ❓ Q6: 工作流出错了怎么办？

**A: Coze有完善的错误处理机制。**

**常见错误及解决**:

| 错误现象 | 可能原因 | 解决方法 |
|---------|---------|---------|
| 意图识别输出不是JSON | Prompt没遵守格式 | 强化输出格式约束 |
| 路由全走到默认分支 | 分类不准确 | 增加训练样本，降低温度 |
| 某个专科回答质量差 | Prompt不够详细 | 参考本手册的完整Prompt |
| 响应超时 | 模型太慢 | 换更快的模型或减少Prompt长度 |
| API调用报401 | Token过期或权限不足 | 重新生成Token |

**调试技巧**:
- 使用 **「预览与调试」** 面板的 **「日志」** 功能
- 查看每个节点的 **输入/输出** 数据
- 定位到具体是哪个节点出了问题
