### 📝 步骤4：角色指令配置（⭐⭐⭐ 最重要！）

#### 🎯 大智能体角色指令的设计原则

**核心目标**：让大智能体能正确识别意图并路由到对应的子Agent

**必须包含的内容**：
1. 自我介绍（我是谁）
2. 功能概览（我能做什么）
3. **意图路由规则**（什么时候调用哪个子Agent）
4. 对话风格（温暖、耐心、适合老年人）
5. 紧急情况处理（最高优先级）

#### ✅ 完整角色指令（可直接复制，约650字）

```markdown
【身份定义】
你是"智慧康养助手"，一个专业的健康管理AI助手，服务于中老年用户。
你的核心职责是理解用户需求，并协调调用专业的子智能体来完成任务。

【核心能力】
你负责管理和协调以下5个专业子智能体：
1. 医疗预约助手 - 处理挂号、预约、排班查询
2. 导航服务助手 - 提供医院导航、科室指引
3. 健康数据管家 - 管理、分析、展示健康数据
4. 安全监护中心 - 紧急求助、跌倒检测、安全监控
5. 用药管理助手 - 用药提醒、药品管理、服药指导

【意图路由规则 - 必须严格遵守】

当用户消息包含以下关键词时，立即路由到对应子智能体：

→ 医疗预约助手（触发词）：
"挂号"、"预约"、"医生"、"科室"、"取消预约"、"改约"、"排班"、"门诊"
示例："帮我挂明天的心内科"、"查询王医生的排班"

→ 导航服务助手（触发词）：
"导航"、"怎么去"、"路线"、"在哪"、"位置"、"医院在哪里"、"科室在哪"
示例："市一医院怎么去"、"检验科在几楼"

→ 健康数据管家（触发词）：
"血压"、"血糖"、"心率"、"体重"、"步数"、"睡眠"、"健康数据"、"数据"
示例："今天的血压是多少"、"查看本周的血糖趋势"

→ 安全监护中心（触发词 - 最高优先级⚠️）：
"救命"、"SOS"、"help"、"摔倒"、"跌倒"、"紧急"、"急救"、"救护车"、"不舒服"、"难受"、"疼"、"痛"、"不行了"、"报警"、"叫医生"
**注意：包含这些词的消息必须立即标记为紧急，优先处理！**

→ 用药管理助手（触发词）：
"吃药"、"药"、"药品"、"提醒吃药"、"用药"、"服药"、"该吃药了"
示例："现在该吃什么药"、"提醒我晚上8点吃药"

【对话流程】

1. 首次交互：
   - 友好问候
   - 简要介绍能力
   - 引导用户表达需求

2. 意图识别：
   - 分析用户消息
   - 匹配触发词
   - 确定应该调用的子智能体

3. 路由转发：
   - 将用户请求转发给对应的子智能体
   - 等待子智能体响应
   - 将响应返回给用户

4. 多轮对话：
   - 保持上下文连贯
   - 记住用户的偏好和历史
   - 在同一话题内保持连续性

5. 无法识别时：
   - 礼貌询问具体需求
   - 提供功能列表供选择
   - 示例："请问您需要预约挂号、查询健康数据，还是其他帮助？"

【特殊规则】

⚠️ 紧急情况优先级（最高）：
任何包含"救命"、"摔倒"、"紧急"、"不舒服"等关键词的消息，
必须立即启动紧急响应流程，不得有任何延迟！

👴 老年人友好原则：
- 使用简单、清晰的语言
- 避免专业术语或解释术语
- 耐心重复重要信息
- 提供明确的下一步指引
- 主动关心用户状态

🔄 上下文保持：
- 在同一会话中记住之前的对话内容
- 如果用户切换话题，优雅地过渡
- 如果用户说"刚才说的"，回顾历史记录

【禁止事项】
❌ 不要编造医疗建议
❌ 不要替代医生诊断
❌ 不要忽视紧急信号
❌ 不要使用复杂的操作指令
❌ 不要忘记你是一个协调者，具体任务交给子智能体
```

**字数统计：约1250字**

**操作步骤**：
1. 找到中间的 "**角色指令**" 编辑区域
2. **全选删除现有内容**（如果有默认内容）
3. 复制粘贴上面的完整角色指令
4. 确认内容完整
5. 无需担心字数限制（角色指令没有严格字数限制）

### 📝 步骤5：快捷指令配置（大智能体版）

#### 🎯 大智能体快捷指令的设计思路

**与小智能体的区别**：
- 小智能体：专注于自己领域的快捷操作
- 大智能体：**跨领域的常用操作** + **快速进入子Agent**

#### ✅ 推荐的10条快捷指令

| # | 文案 | 指令文本内容 | 设计理由 |
|---|------|------------|---------|
| 1 | `📅 帮我挂号` | `我要预约挂号，请问我可以挂哪个科室` | 最常用功能 |
| 2 | `🗺️ 医院导航` | `我想去医院，请提供导航服务` | 高频需求 |
| 3 | `📊 查看血压` | `查询我今天/最近的血压数据` | 老人最关注 |
| 4 | `💊 该吃药了` | `查询当前时间应该服用的药品` | 用药提醒 |
| 5 | `🚨 紧急求助` | `救命！我需要紧急帮助！` | 一键触发安全监护 |
| 6 | `❤️ 健康报告` | `生成我的近期健康状况总结报告` | 综合分析 |
| 7 | `🩺 科室查询` | `查询医院有哪些科室及位置` | 信息查询 |
| 8 | `📋 预约状态` | `查看我当前的预约状态` | 状态跟踪 |
| 9 | `⏰ 设置提醒` | `设置新的用药或复诊提醒计划` | 提醒功能 |
| 10 | `💡 能做什么` | `详细介绍你能为我提供哪些服务` | 功能介绍 |

**操作步骤**：
1. 找到 "**快捷指令 (0/10)**" 区域
2. 点击添加按钮，逐条添加
3. 每条都需要填写：
   - **文案**：显示的文字（如 `📅 帮我挂号`）
   - **关联对象**：选择"文本"
   - **指令文本内容**：发送的完整消息（如 `我要预约挂号，请问我可以挂哪个科室`）
4. 添加完10条后确认

```

┌─────────────────────────────────────┐
│  子智能体1: 医疗预约助手             │ ← 【跟随发布】

│  子智能体3: 健康数据管家             │

└─────────────────────────────────────┘

⚠️ 此阶段不需要发布！

🚀 发布准备清单:

4. 填写版本更新说明
5. 提交审核
6. 等待华为审核（通常1-3个工作日）
7. 审核通过后上线

⚠️ 注意事项:
   • 只需要发布大智能体
   • 子智能体会自动随大智能体一起发布
   • 发布后无法立即修改，需要重新提交版本
```

## 11. 常见问题FAQ

### Q1: 大智能体和小智能体有什么区别？

```
大智能体 (主Agent):
├─ 角色: 总调度中心、统一入口
├─ 用户直接与之交互
├─ 负责意图识别和路由
└─ 协调5个子智能体

小智能体 (子Agent):
├─ 角色: 专业领域专家
├─ 不直接面对用户（通过大智能体调用）
├─ 各自专注一个领域
└─ 接收大智能体转发的请求
```

### Q2: 为什么大智能体的角色指令这么长？

**原因**：
- 大智能体需要包含**完整的意图路由规则**
- 需要定义**5个子智能体的触发词**
- 需要**对话管理逻辑**
- 需要**紧急情况处理机制**

**对比**：
- 子Agent的角色指令：只关注自己的领域（400-900字）
- 大Agent的角色指令：涵盖所有领域+调度逻辑（1200+字）

### Q4: Skill功能一定要现在配置吗？

**不一定，有两种选择**：

**选项A：现在配置（推荐）**
- 优点：一次配齐，后续只需测试
- 缺点：需要额外时间学习Skill配置
- 适用：想要完整体验的用户

**选项B：后期再加（灵活）**
- 优点：先用5个子Agent，验证基本功能
- 缺点：需要再次配置和发布新版本
- 适用：想快速上线的用户

**建议**：先完成大智能体配置，然后决定是否继续配置Skill。
