🚨 报错原因 + 100%解决办法
这个错误非常明确：
plaintext
UNIQUE constraint failed: user_info.phone
翻译：手机号重复了，违反唯一约束，插入失败！
1. 为什么报错？
你在建表时写了：
sql
phone TEXT UNIQUE NOT NULL
UNIQUE = 唯一约束意思是：同一个手机号，数据库里只能存一条，不能重复插入
你现在反复执行：
sql
INSERT INTO user_info (username, phone) VALUES ('zhangsan', '13900139000');
第一次能插入，第二次开始就会报错，因为号码已经存在了！
2. 立刻解决（3 种方法任选）
方法 1：换一个新手机号（最简单）
把号码随便改一下，不重复就能插入：
sql
INSERT INTO user_info (username, phone) VALUES ('lisi', '13812345678');
方法 2：先查询看看库里已经有什么号码
sql
SELECT * FROM user_info;
看到已有的号码，不要再用它。
方法 3：删除重复数据，重新插入
sql
DELETE FROM user_info WHERE phone = '13900139000';
然后再执行插入就不会报错了。
