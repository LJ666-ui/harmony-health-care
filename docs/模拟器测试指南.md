# 模拟器测试指南

## 问题背景

在HarmonyOS模拟器上测试医学影像AI分析功能时，可能会遇到以下问题：
1. 模拟器没有摄像头，无法使用拍照功能
2. 模拟器相册中没有图片，无法从相册选择
3. 无法测试AI影像分析功能

## 解决方案

我们提供了**示例图片功能**，让你可以在模拟器环境下完整测试AI影像分析功能。

### 使用方法

1. 打开"医学影像AI分析"页面
2. 选择你想要测试的影像类型（如：胸部X光、骨骼X光等）
3. 点击**"使用示例图片测试"**按钮（橙色按钮）
4. 系统会自动加载对应类型的示例图片
5. 点击"开始分析"按钮，即可测试AI分析功能

### 功能特点

- ✅ 无需摄像头
- ✅ 无需相册图片
- ✅ 支持5种影像类型
- ✅ 完整测试AI分析流程
- ✅ 适用于模拟器和真机

## 技术实现

### 核心文件

1. **ImageAnalysisPage.ets** - 主页面，添加了示例图片按钮
2. **SampleImageUtil.ets** - 示例图片管理工具类
3. **imagingMock.ets** - Mock数据，提供模拟分析结果

### 代码示例

```typescript
// 使用示例图片
private async useSampleImage() {
  this.imageUri = SampleImageManager.getSampleImagePath(this.selectedImagingType);
}

// 获取示例图片路径
static getSampleImagePath(imagingType: ImagingType): string {
  const sampleImage = this.getSampleImageByType(imagingType);
  if (sampleImage) {
    return sampleImage.resourcePath;
  }
  return this.getPlaceholderImage(imagingType);
}
```

## 其他测试方案

### 方案1：使用网络图片

如果模拟器有网络连接，可以直接使用网络图片URL：

```typescript
this.imageUri = 'https://example.com/medical-image.jpg';
```

### 方案2：使用Base64图片

将图片转换为Base64编码直接嵌入：

```typescript
this.imageUri = 'data:image/jpeg;base64,/9j/4AAQSkZJRg...';
```

### 方案3：ADB推送图片

使用ADB命令向模拟器推送图片：

```bash
adb push test-image.jpg /sdcard/Pictures/
```

然后就可以在相册中看到图片了。

## 常见问题

### Q: 示例图片无法显示？

A: 检查以下几点：
1. 网络连接是否正常（如果使用网络图片）
2. 图片URL是否正确
3. 查看控制台日志是否有错误信息

### Q: 如何添加更多示例图片？

A: 修改 `SampleImageUtil.ets` 中的 `getAllSampleImages()` 方法，添加更多图片信息。

### Q: 真机上可以使用吗？

A: 可以！示例图片功能在真机上同样有效，可以用于快速演示和测试。

## 总结

通过示例图片功能，你可以在任何环境下（模拟器或真机）完整测试医学影像AI分析功能，无需依赖摄像头或相册。这是最简单、最直接的测试方案！
