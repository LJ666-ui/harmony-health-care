**关键技术实现**：

### 5.3 RAG 检索增强技术

**混合检索策略** (HybridRetriever)：
```typescript
interface HybridConfig {
  vectorWeight: number;      // 向量检索权重（默认 0.7）
  keywordWeight: number;     // 关键词检索权重（默认 0.3）
  useRerank: boolean;        // 是否启用重排序
  topK: number;              // 返回结果数量
}
```

**检索流程**：
```
查询文本
  ├── VectorRetriever.search(query, topK)
  │     → embed(query) 向量化
  │     → cosineSimilarity() 余弦相似度计算
  │     → 返回 TopK 向量检索结果
  │
  ├── KeywordRetriever.search(query, topK)
  │     → 关键词匹配 + TF-IDF 打分
  │     → 返回 TopK 关键词检索结果
  │
  └── 结果合并 + 加权评分
        → vectorWeight × vectorScore + keywordWeight × keywordScore
        → 如果 useRerank: Reranker.rerank() 精确重排序
        → 返回最终结果
```

**向量检索配置**：
```typescript
interface VectorConfig {
  embeddingModel: 'bge-large-zh';   // 中文嵌入模型
  dimension: 1024;                   // 向量维度
  indexType: 'faiss';                // 索引类型
}
```

**AutomationEngine** 技术实现：

```typescript
// 规则模型
interface AutomationRule {
  ruleId: string;
  ruleName: string;
  ruleType: RuleType;           // 规则类型
  triggerConditions: ConditionGroup;  // 触发条件组
  actions: Action[];            // 执行动作
  enabled: boolean;
  cooldown: number;             // 冷却时间
}

// 触发条件
interface ConditionGroup {
  logic: 'AND' | 'OR';         // 逻辑关系
  conditions: RuleCondition[];  // 条件列表
}

// 触发类型
enum TriggerType {
  DEVICE_STATUS,    // 设备状态触发
  TIME_BASED,       // 定时触发
  THRESHOLD,        // 阈值触发
  EVENT_BASED       // 事件触发
}
```

**规则执行流程**：
```
事件发生 → AutomationEngine.evaluateRules(event)
  → 遍历所有已启用规则
  → 检查冷却时间（cooldownMap）
  → 评估触发条件（ConditionGroup 逻辑运算）
  → RuleExecutor.execute(rule, event)
  → 执行动作列表
  → 返回 RuleExecutionResult
```
