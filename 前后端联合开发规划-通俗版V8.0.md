**⚠️ 数据依赖**: Day 12必须完成图谱数据构建（40-50节点+80-100边），否则Canvas画布空白！

**技术难点预判**:

```
难点1: Canvas渲染性能
  问题: 50个节点+100条边同时绘制可能卡顿
  解决:
    ├── 分层渲染: 先画边(底层)，再画节点(顶层)
    ├── 虚拟化: 只视口内节点参与绘制
    ├── 缓存: 静态内容缓存为OffscreenCanvas
    └── 限制: 节点数>100时提示"数据量大，建议缩小范围"

难点2: 力导向布局算法
  问题: 节点初始位置随机，需要物理模拟排布
  解决:
    ├── 使用简化的Fruchterman-Reingold算法
    ├── 迭代次数限制≤50次（避免卡顿）
    ├── 初始温度参数调优（影响扩散速度）
    └── 固定中心节点位置（centerId对应的节点居中）

难点3: 交互事件处理
  问题: Canvas点击检测（判断点击了哪个圆形节点）
  解决:
    ├── 维护节点坐标数组 [{id, x, y, radius}]
    ├── click事件 → 计算距离 → 判断是否在半径内
    ├── 支持触摸事件 (touchstart/touchmove/touchend)
    └── 双击检测 (两次click间隔<300ms)

难点4: 手势缩放平移
  问题: 手机上单指拖拽+双指缩放
  解决:
    ├── 监听 touchmove 事件
    ├── 单指: 计算偏移dx/dy → 平移整个坐标系
    ├── 双指: 计算两指距离变化 → 缩放scale因子
    └── 矩阵变换: ctx.transform(scale, 0, 0, scale, tx, ty)
```

║  │   ├── 数据类型 (String/Integer/List匹配)                   ║
║  │   └── 边界场景 (空数据/非法参数/超长字符串)                 ║

║  │   ├── const res = await HttpUtil.get(url, params)         ║

║  ├── Model映射 (TypeScript interface定义)                    ║
║  ├── 页面UI渲染:                                             ║

║  │   ├── EmptyState (空数据时友好提示)                        ║
║  │   └── ErrorComponents (网络错误/服务器错误)                ║
║  └── 边界情况处理:                                            ║
│      ├── 分页到底了 (hasMore=false)                           ║
