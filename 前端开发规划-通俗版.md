##### **HttpUtil.ets** - HTTP请求封装（🔴🔴🔴 极其重要！）

**作用**：前端与后端通信的**唯一通道**

**核心能力**：
- 发送GET/POST/PUT/DELETE请求
- 自动携带JWT Token到请求头
- 统一错误处理（401自动跳转登录）
- 响应拦截器（统一解析Result格式）

**使用示例**：
```typescript
import { http } from '@kit.NetworkKit';
import { ApiConstants } from './constants/ApiConstants';

export class HttpUtil {
  
  static async post(url: string, data: object): Promise<any> {
    const token = AppStorage.get<string>('token');
    
    const response = await http.createRequest()
      .url(ApiConstants.BASE_URL + url)
      .method(http.RequestMethod.POST)
      .header({
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${token}`  // 自动携带Token
      })
      .data(JSON.stringify(data));
    
    const result = await response.result();
    
    if (result.code === 401) {
      // Token过期，跳转登录
      router.pushUrl({ url: 'pages/Login' });
    }
    
    return result;
  }
}

// 在页面中使用
async handleLogin() {
  const result = await HttpUtil.post('/user/login', {
    username: this.username,
    password: this.password
  });
  
  if (result.code === 200) {
    AppStorage.setOrCreate<string>('token', result.data.token);
    router.pushUrl({ url: 'pages/HomePage' });
  } else {
    AlertDialog.show({ message: result.message });
  }
}
```

#### **4️⃣ store/AppStore.ets** - 全局状态管理

**作用**：类似Vue的Vuex或React的Redux，跨页面共享数据

**存储内容**：
- 当前登录用户信息
- JWT Token
- 适老化模式开关状态
- 全局设置项

**工作流程**：
```
1. 登录成功 → 存储用户信息和Token到AppStore
2. 其他页面 → 直接从AppStore读取（无需再次请求后端）
3. 退出登录 → 清空AppStore，跳转登录页
4. 切换适老化模式 → 更新AppStore，全局UI自动刷新
```

### 🎨 适老化设计规范（你必须遵守！）

| 设计要素 | 标准要求 | 你的实现方式 |
|----------|---------|-------------|
| **字体大小** | 正文≥16px，标题≥20px | 使用GlobalTheme.getFontSizeBody(isElderMode) |
| **按钮尺寸** | 最小高度44px，最小宽度44px | Button组件已内置此规则 |
| **触摸目标** | 间距至少8px | Card组件的padding和margin |
| **颜色对比度** | 至少4.5:1（WCAG AA标准） | GlobalTheme中定义的高对比色 |
| **行高** | 至少1.5倍字号 | 样式文件中的lineHeight设置 |
| **图标大小** | 至少24x24px | Footer组件的大图标 |

**如何切换适老化模式**：
```typescript
@StorageLink('isOldModeEnabled') isElderMode: boolean = false;

// 在build()中使用
Text('欢迎回来')
  .fontSize(GlobalTheme.getFontSizeTitle(this.isElderMode))  // 自动适配
```

### 🗓️ Day 1：网络请求模块+登录注册（6小时）⭐最重要的一天

**具体任务**：

**第一部分：HTTP请求基础设施（3小时）**

   - 统一错误处理（401跳转登录、500友好提示）
   - 请求超时10秒、显示加载状态

2. **创建请求/响应拦截器**（45分钟+30分钟）
   - 请求前检查Token，过期自动清除跳转登录
   - 统一解析后端返回格式，错误码映射

3. **测试验证**（15分钟）
   - 确认后端服务正常启动

**第二部分：改造登录页面（2小时）**

4. **改造Login.ets**（1.5小时）
   - 点击登录调用后端 /user/login 接口
   - 成功后保存Token到本地存储
   - 失败显示错误信息（密码错、账号不存在等）
   - 加载中状态防重复提交

5. **创建Register.ets注册页**（30分钟）
   - 手机号+密码+确认密码表单
   - 手机号校验（11位数字）、密码强度检测
   - 两次密码一致性校验
   - 调用后端注册接口，成功跳转登录

**第三部分：Token管理和路由守卫（1小时）**

6. **创建AuthManager.ets认证管理器**（45分钟）
   - 保存/读取/删除Token
   - 检查是否已登录、过期判断

## ⚠️ 六、风险预案

**风险1：后端接口还没准备好**（概率50%）⚠️ 高概率
- 先用Mock数据开发前端UI
- 定义好接口文档，等后端好了再替换
- Day1就和后端C对齐接口格式

**风险2：3D模型加载失败或不支持**（概率40%）
- 准备降级方案：用图片序列或视频代替
- 不影响核心功能，只是体验差一点
- 提前测试ArkUI的3D能力

**风险3：AI接口响应慢或失败**（概率35%）
- 前端做好loading状态和超时处理
- 准备本地预设答案作为降级
- 缓存常见问题的回答

**风险4：适老化需求理解不一致**（概率30%）
- Day7就做适老化全面测试
- 找真实老年用户试用（家人/邻居）
- 收集反馈快速调整

**风险5：联调时发现大量问题**（概率45%）
- 预留Day18-19两天专门联调
- 问题按优先级分类处理
- P0当天解决，P1尽量解决，P2记录

**风险6：某个页面超时**（概率35%）
- 每晚评估进度及时发现延期
- 优先保证P0任务
- P2任务可简化或砍掉
- 复杂动画可暂时省略

## 💡 八、前端开发特别注意事项

### 适老化设计原则（贯穿始终）

1. **字体大小**：正文至少16px，标题至少20px，老年模式至少20%/30%/50%放大
2. **按钮尺寸**：最小触控区域44x44px，老年模式56x56px
3. **颜色对比度**：文字和背景对比度至少4.5:1（AA标准）
4. **间距宽松**：元素之间留足空间，避免拥挤
5. **减少层级**：页面不超过3层跳转，避免迷路
6. **明确反馈**：每次操作都有明确的视觉或震动反馈
7. **容错性强**：允许撤销操作，确认后再执行危险操作
8. **语音辅助**：重要信息支持TTS语音播报

### 与后端协作要点

1. **提前对齐接口格式**：Day1就和后端C约定好数据结构
2. **使用真实数据测试**：不要只用Mock数据，尽早用真实接口
3. **错误码统一**：前后端错误码含义一致
4. **及时沟通**：发现接口问题立刻联系后端C，不要自己猜测
5. **版本兼容**：考虑接口升级时的兼容性

### 性能优化建议

1. **图片优化**：压缩图片、使用WebP格式、懒加载
2. **列表优化**：长列表使用LazyForEach虚拟滚动
3. **请求优化**：合并请求、避免N+1问题、合理缓存
4. **包体积**：按需引入组件、移除无用代码、资源压缩

      ```typescript
      // 核心方法
      async get<T>(url: string, params?: object): Promise<Result<T>>
      async post<T>(url: string, data?: object): Promise<Result<T>>
      async put<T>(url: string, data?: object): Promise<Result<T>>
      async delete<T>(url: string): Promise<Result<T>>

      - 自动读取Token并附加到请求头（Authorization: Bearer xxx）
      - 统一响应格式解析：{code: 200, message: "成功", data: T}

      - 响应拦截器：
        * code=200 → 返回data
        * code=401 → 未授权 → 清除Token → 跳转登录
        * code=400 → 参数错误 → 显示message提示
        * code=500 → 服务器错误 → 显示友好提示
        * 网络异常 → 显示"网络连接失败，请检查网络"
      - 请求超时设置：10秒
      - 统一错误处理：Toast提示 + 日志记录
      - 加载状态管理：请求开始显示Loading，结束隐藏
      ```

      ```typescript
      @Observable
      class HealthViewModel {
        records: HealthRecord[] = []
        currentPage: number = 1
        hasMore: boolean = true
        selectedType: string = 'blood_pressure'
        filterStatus: string = 'all'
        
        async loadRecords(refresh: boolean = false): Promise<void> {
          if (refresh) this.currentPage = 1
          
          const result = await HttpUtil.get<PageResult<HealthRecord>>(
            '/health/records',
            {
              type: this.selectedType,
              status: this.filterStatus,
              page: this.currentPage,
              size: 20
            }
          )
          
          if (refresh) {
            this.records = result.data.list
          } else {
            this.records = [...this.records, ...result.data.list]
          }
          
          this.hasMore = this.records.length < result.data.total
          this.currentPage++
        }
        
        async addRecord(record: Partial<HealthRecord>): Promise<boolean> {
          const result = await HttpUtil.post('/health/record', record)
          if (result.code === 200) {
            await this.loadRecords(true)  // 刷新列表
            return true
          }
          return false
        }
        
        async deleteRecord(id: number): Promise<boolean> {
          const result = await HttpUtil.delete(`/health/record/${id}`)
          if (result.code === 200) {
            this.records = this.records.filter(r => r.id !== id)
            return true
          }
          return false
        }
        
        loadMore(): void {
          if (this.hasMore && !this.isLoading) {
            this.loadRecords(false)
          }
        }
      }
      ```
