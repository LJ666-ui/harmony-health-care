## 九、常见问题FAQ

### 9.1 开发环境问题

**Q1: DevEco Studio安装后无法启动？**
A: 检查以下几点：
1. JDK版本是否为JDK 17（HarmonyOS要求）
2. 环境变量JAVA_HOME是否正确配置
3. 是否有足够的磁盘空间（建议>50GB）
4. 是否安装了必要的SDK（API 9+）
5. 尝试删除用户配置目录重新启动：C:\Users\xxx\.devecostudio

**Q2: 项目编译报错"Module not found"？**
A: 解决步骤：
1. 关闭DevEco Studio
2. 删除项目目录下的.oh_modules文件夹
3. 删除node_modules文件夹
4. 重新打开项目
5. 等待依赖自动下载完成
6. 如果还是失败，检查网络是否需要代理

**Q3: 真机调试连接不上？**
A: 检查以下几点：
1. USB数据线是否支持数据传输（不是只能充电）
2. 手机是否开启"开发者模式"
3. 手机是否开启"USB调试"
4. 是否信任此电脑（手机弹出提示时点击允许）
5. 驱动是否正确安装（华为手机助手可自动安装驱动）
6. hdc list targets命令能否看到设备

**Q4: 签名配置错误怎么办？**
A: 解决步骤：
1. 确保已申请华为开发者账号
2. 在AppGallery Connect创建应用并获取证书
3. DevEco Studio → File → Project Structure → Signing Configs
4. 勾选"Automatically generate signature"
5. 或手动选择.p12证书文件和.csr文件
6. Clean Project后重新Build

---

### 9.2 运行时问题

**Q5: 应用启动白屏/黑屏？**
A: 可能原因及解决方案：
1. EntryAbility.ets中onWindowStageCreate()未正确加载首页
2. main_pages.json中的路由配置有误
3. Index.ets页面加载过慢（检查网络请求/图片加载）
4. 检查HiLog是否有报错信息
5. 尝试在Index.ets添加Loading状态提示

**Q6: 页面跳转失败？**
A: 常见原因：
1. router.pushUrl()中的url路径错误（需与main_pages.json一致）
2. 目标页面不存在或路径拼写错误
3. 页面参数传递格式错误
4. 检查路由守卫（如需要登录但Token过期）

**Q7: 网络请求失败？**
A: 排查步骤：
1. 检查网络权限：module.json5中是否有ohos.permission.INTERNET
2. 检查后端服务是否启动：浏览器访问http://localhost:8080/swagger-ui.html
3. 检查API地址是否正确（开发环境vs生产环境）
4. 检查跨域配置（CORS）
5. 使用Postman/Swagger先验证接口可用性
6. 查看HttpUtil的错误回调输出详细信息

**Q8: SQLite数据库操作失败？**
A: 常见原因：
1. 数据库文件路径错误
2. SQL语句语法错误（字段名/表名/数据类型）
3. 数据库未打开就执行操作
4. 并发读写冲突（使用事务包裹）
5. 检查RdbHelper的初始化是否成功

---

### 9.3 UI/UX问题

**Q9: 页面布局在不同屏幕尺寸显示异常？**
A: 解决方案：
1. 使用相对布局而非绝对定位（Flex/Row/Column）
2. 使用百分比或vp单位而非固定px
3. 测试不同分辨率模拟器（3.5"/5.5"/7.0"）
4. 使用媒体查询适配不同屏幕：
   ```typescript
   // 响应式布局示例
   .width(this.isLargeScreen ? '80%' : '100%')
   ```

**Q10: 列表滑动卡顿？**
A: 优化方案：
1. 使用LazyForEach替代ForEach（懒加载）
2. 减少列表项复杂度（简化组件嵌套）
3. 图片使用异步加载+缓存
4. 避免在滑动时进行耗时操作
5. 开启虚拟滚动（长列表必备）

**Q11: 字体大小切换后布局乱掉？**
A: 解决方案：
1. 使用vp单位而非fp单位（vp会随系统字号缩放）
2. 限制最大字号（避免过大导致溢出）
3. 测试1x/1.25x/1.5x/1.75x/2x五种字号
4. 关键容器设置overflow隐藏
5. 使用弹性布局自适应

---

### 9.4 性能问题

**Q12: 应用启动慢？**
A: 优化方案：
1. 减少EntryAbility初始化逻辑（延迟加载非关键模块）
2. 图片资源压缩（WebP格式比PNG小50%+）
3. 预加载关键数据（首屏数据优先）
4. 启动页增加品牌展示（转移用户注意力）
5. 使用Release构建（移除调试代码+启用混淆）

**Q13: 内存占用过高？**
A: 排查与优化：
1. 使用Profiler工具监控内存曲线
2. 检查是否有内存泄漏（定时器/监听器未释放）
3. 图片及时释放（不再显示时置空）
4. 列表数据分页加载（不要一次加载全部）
5. 使用@Cached装饰器缓存计算结果
6. aboutToDisappear()生命周期中清理资源

**Q14: 电池耗电快？**
A: 优化方案：
1. 减少后台定位请求频率
2. 合并网络请求（批量上传代替逐条上传）
3. 使用WebSocket代替轮询
4. 图片下载完成后停止网络请求
5. 合理使用@StorageLink/@LocalStorage减少状态更新频率

---

### 9.5 发布上线问题

**Q15: 应用上架被拒？**
A: 常见原因：
1. 缺少隐私政策链接（必须提供）
2. 权限申请理由不充分
3. 应用描述与实际功能不符
4. 包含违规内容（广告/诱导下载）
5. 未通过兼容性测试
6. 解决：仔细阅读华为应用市场审核指南

**Q16: 用户反馈闪退？**
A: 排查步骤：
1. 收集崩溃日志（华为AppGallery Connect → 崩溃服务）
2. 定位崩溃堆栈（找到具体代码行）
3. 分析崩溃原因（空指针/数组越界/类型错误）
4. 修复Bug后尽快发版（热修复或正常更新）
5. 建立Crashlytics等崩溃监控系统
