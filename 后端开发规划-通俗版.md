# 一、项目是什么？

## 1.1 一句话介绍

**星云医疗助手** = 一个专门给中老年人用的**健康管理手机App**的后台服务系统

## 1.2 打个比方

想象一下这个系统就像一个**医院的信息中心**：

```
┌─────────────────────────────────────────────────────┐
│                    星云医疗助手                        │
│                                                       │
│  前端（App界面）          后端（服务器大脑）             │
│  ┌───────────┐         ┌──────────────────┐           │
│  │ 用户看到的  │  ←──→   │ 处理所有业务逻辑    │           │
│  │ 手机屏幕    │         │ 存储所有数据       │           │
│  └───────────┘         └──────────────────┘           │
│         ↑                      ↑                       │
│    老年人操作简单           数据安全可靠                │
│                                                       │
└─────────────────────────────────────────────────────┘
```

**前端（你们已经做好的）**: 
- 24个页面（登录、首页、医院、预约、健康记录、康复、科普等）
- 使用鸿蒙系统（HarmonyOS）开发
- 已经可以展示Mock数据（假数据）

**后端（我们现在要做的）**:
- 用Java语言开发（Spring Boot框架）
- 连接MySQL数据库存储真实数据
- 提供API接口给前端调用
- 确保数据安全（加密、权限控制）

# 二、后端要做什么？（8大功能模块）

根据数据库的22张表，我们把后端功能分成**8个独立模块**：

## 📦 模块1：用户管理（相当于"门卫"）

**作用**: 管理谁能用这个App，保护用户信息安全

### 要实现的功能：
| 功能 | 通俗解释 | 对应数据库表 |
|------|---------|-------------|
| ✅ 用户注册 | 新用户创建账号（手机号+密码） | `user` 表 |
| ✅ 用户登录 | 验证身份，发一张"通行证"（JWT Token） | `user` 表 |
| ✅ 修改密码 | 用户想改密码 | `user` 表 |
| ✅ 上传头像 | 更换个人照片 | `user` 表 |
| ✅ 设备绑定 | 绑定用户的手机/手表/平板等设备 | `harmony_device` 表 |

**打个比方**: 就像小区门卫，验证你是业主才让你进，还给你发门禁卡。

---

## 📦 模块2：医疗服务（相当于"挂号窗口"）

| ✅ 电子病历 | 记录每次看病的诊断和治疗方案 | `medical_record` 表 |

```
┌─────────────────────────────────────────────────────────────┐

│                                                               │
│  ┌─────────────────────────────────────────────────────┐     │
│  │              接口层（Controller）                    │     │
│  │  相当于"前台接待员"，接收前端的请求                   │     │

│                         │                                    │
│  ┌──────────────────────▼──────────────────────────────┐     │

│  ┌─────────────────────────────────────────────────────┐     │
```

## 3.3 各层的职责说明（用餐厅打比方）

| 层级 | 名称 | 餐厅比喻 | 具体工作 |
|------|------|---------|---------|
| **Controller** | 控制器层 | **服务员** | 接收顾客订单（前端请求），转交给厨师 |
| **Service** | 业务逻辑层 | **厨师** | 按照菜谱（业务规则）加工食材（数据） |
| **Mapper** | 数据访问层 | **采购员** | 从仓库（数据库）取食材，或把剩菜放回去 |
| **Entity** | 实体类 | **菜单卡片** | 描述每道菜的成分（表的字段） |
| **DTO** | 数据传输对象 | **订单小票** | 记录顾客点了什么菜（请求/响应数据） |
| **Config/Utils** | 配置工具 | **厨房设备** | 炒锅、刀具、调料（公共工具） |

**数据流转过程**：
```
前端发送请求 → Controller接收 → Service处理业务 → Mapper操作数据库 → 返回结果
   (顾客点餐)    (服务员接单)    (厨师做菜)      (采购员取材)     (上菜)
```

**影响**：B无法按时联调，D无法完成测试

**解决方案**：

   - 复杂查询（统计/趋势）可以简化实现
   - 单元测试覆盖率可以从60%降到40%
4. **调整联调顺序**：B先联调已完成的核心模块（用户+健康+预约），P2模块最后再做

2. **引入"已知问题清单"**：
   - 在答辩时主动说明已知问题和改进方向

## 9.2 代码规范（A制定，全员遵守）

### Java代码规范（后端）

```java
// ✅ 正确的命名示例
public class UserServiceImpl implements UserService {  // 类名：大驼峰
    private UserMapper userMapper;                       // 成员变量：小驼峰
    
    /**
     * 用户登录
     * @param loginRequest 登录请求参数
     * @return 用户信息和Token
     */
    public Result<UserInfoResponse> login(LoginRequest loginRequest) {  // 方法名：小驼峰
        // 1. 参数校验
        if (StringUtils.isBlank(loginRequest.getPhone())) {
            return Result.error("手机号不能为空");
        }
        
        // 2. 业务逻辑
        User user = userMapper.selectByPhone(loginRequest.getPhone());
        if (user == null) {
            return Result.error("用户不存在");
        }
        
        // 3. 返回结果
        UserInfoResponse response = convertToDTO(user);
        String token = jwtUtil.generateToken(user.getId());
        return Result.success(response, token);
    }
}
```

**关键规范**：
- ✅ 类名使用大驼峰（PascalCase）
- ✅ 方法和变量使用小驼峰（camelCase）
- ✅ 常量使用全大写下划线（MAX_RETRY_COUNT）
- ✅ 每个公有方法必须有JavaDoc注释
- ✅ 关键业务逻辑要写注释解释"为什么"（不是"做什么"）
- ✅ 一个方法不超过50行（太长就拆分）
- ✅ 避免深度嵌套（if超过3层就用early return）

```typescript
// ✅ 正确的组件示例
@Component
struct HealthRecordCard {
  @Prop record: HealthRecord = {} as HealthRecord;
  @State isElderMode: boolean = false;
  
  build() {
    Column() {
      // 标题
      Text(this.record.type)
        .fontSize(GlobalTheme.getFontSizeTitle(this.isElderMode))
        .fontColor(GlobalTheme.TEXT_PRIMARY)
      
      // 数值
      Text(`${this.record.value} ${this.record.unit}`)
        .fontSize(GlobalTheme.getFontSizeBody(this.isElderMode))
        .fontColor(GlobalTheme.THEME_COLOR)
    }
    .width('100%')
    .padding(GlobalTheme.getSpacingMD(this.isElderMode))
    .backgroundColor(GlobalTheme.CARD_BG)
    .borderRadius(GlobalTheme.RADIUS_MD)
  }
}
```

**关键规范**：
- ✅ 组件名使用大驼峰（PascalCase）
- ✅ 变量和函数使用小驼峰（camelCase）
- ✅ 必须使用GlobalTheme统一样式（禁止硬编码fontSize/fontColor等）
- ✅ Props必须设置默认值（避免undefined报错）
- ✅ State变量在aboutToAppear中初始化
