### 2.1 用户认证模块（6个API）⭐基础核心

**模块概述**: 提供用户注册、登录、信息管理等基础认证功能

| 编号 | API名称 | 请求路径 | 方法 | 建什么文件 | 有什么用处 | 参数 | 返回值 | 状态 |
|------|---------|---------|------|-----------|----------|------|--------|------|
| U01 | 用户登录 | /api/user/login | POST | [UserController.java](src/main/java/com/example/medical/controller/UserController.java) | 用户手机号+密码登录，获取JWT Token | phone(手机号), password(密码) | {token, userInfo} | ✅已完成 |
| U02 | 用户注册 | /api/user/register | POST | UserController.java | 新用户创建账号 | phone, password, nickname, code(验证码) | {userId, message} | ✅已完成 |
| U03 | 获取用户信息 | /api/user/info | GET | UserController.java | 根据Token获取当前用户详细信息 | Header: Authorization | UserVO对象 | ✅已完成 |
| U04 | 更新用户信息 | /api/user/update | PUT | UserController.java | 修改昵称/头像/年龄等 | nickname, avatar, age, gender | Result.success() | ✅已完成 |
| U05 | 修改密码 | /api/user/password | PUT | UserController.java | 修改登录密码 | oldPassword, newPassword | Result.success() | ✅已完成 |
| U06 | 退出登录 | /api/user/logout | POST | UserController.java | 注销当前Token，清除Redis缓存 | Header: Authorization | Result.success() | ✅已完成 |

**【详细业务逻辑说明】**

#### U01 用户登录 - 核心流程
```
前端请求 → Controller接收参数 → Service校验:
    ① 检查phone格式(11位数字)
    ② 根据phone查询User表
    ③ BCryptUtil.matches()比对密码
    ④ 生成JWT Token(JwtUtil.generateToken())
    ⑤ Token存入Redis(设置24小时过期)
    ⑥ 返回Token + 用户基本信息

异常处理:
    - 手机号未注册 → 返回404 "用户不存在"
    - 密码错误 → 返回401 "密码错误"
    - 账户被锁定 → 返回403 "账户已被锁定"
```

#### U02 用户注册 - 核心流程
```
前端请求 → Controller接收参数 → Service处理:
    ① 校验验证码是否正确(Redis中取)
    ② 检查phone是否已注册
    ③ BCryptUtil.encrypt()加密密码
    ④ 创建User对象并插入数据库
    ⑤ 自动生成JWT Token(注册即登录)
    ⑥ 返回UserId + Token

数据校验规则:
    - 手机号: 正则^1[3-9]\d{9}$
    - 密码: 8-20位，必须含字母和数字
    - 昵称: 2-20位，不能含特殊字符
```

**【建这些API有什么用】**
- **U01登录**: 整个系统的入口，所有后续请求都依赖Token
- **U02注册**: 新用户入驻的唯一途径
- **U03获取信息**: 个人中心页面展示用户头像、昵称等
- **U04更新信息**: 用户修改个人资料
- **U05改密码**: 安全功能，定期更换密码
- **U06退出登录**: 安全退出，清除服务端会话

```sql
-- MySQL窗口函数计算7天移动平均
SELECT 
    record_time,
    value,
    AVG(value) OVER (
        ORDER BY record_time 
        ROWS BETWEEN 6 PRECEDING AND CURRENT ROW
    ) as ma_7day,
    -- 判断是否在正常范围
    CASE 
        WHEN dataType='blood_pressure' 
             AND CAST(SUBSTRING_INDEX(value,'/',1) AS DECIMAL)>140 
        THEN 'high'
        ELSE 'normal'
    END as status
FROM health_records 
WHERE user_id = #{userId} 
  AND data_type = #{dataType}
  AND record_time >= DATE_SUB(NOW(), INTERVAL #{days} DAY)
ORDER BY record_time ASC;
```
