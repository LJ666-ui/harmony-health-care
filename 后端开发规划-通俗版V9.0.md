### 2.1 用户认证模块（6个API）⭐基础核心

**模块概述**: 提供用户注册、登录、信息管理等基础认证功能

| 编号 | API名称 | 请求路径 | 方法 | 建什么文件 | 有什么用处 | 参数 | 返回值 | 状态 |
|------|---------|---------|------|-----------|----------|------|--------|------|
| U01 | 用户登录 | /api/user/login | POST | [UserController.java](src/main/java/com/example/medical/controller/UserController.java) | 用户手机号+密码登录，获取JWT Token | phone(手机号), password(密码) | {token, userInfo} | ✅已完成 |
| U02 | 用户注册 | /api/user/register | POST | UserController.java | 新用户创建账号 | phone, password, nickname, code(验证码) | {userId, message} | ✅已完成 |
| U03 | 获取用户信息 | /api/user/info | GET | UserController.java | 根据Token获取当前用户详细信息 | Header: Authorization | UserVO对象 | ✅已完成 |
| U04 | 更新用户信息 | /api/user/update | PUT | UserController.java | 修改昵称/头像/年龄等 | nickname, avatar, age, gender | Result.success() | ✅已完成 |
| U05 | 修改密码 | /api/user/password | PUT | UserController.java | 修改登录密码 | oldPassword, newPassword | Result.success() | ✅已完成 |
| U06 | 退出登录 | /api/user/logout | POST | UserController.java | 注销当前Token，清除Redis缓存 | Header: Authorization | Result.success() | ✅已完成 |

**【详细业务逻辑说明】**

#### U01 用户登录 - 核心流程
```
前端请求 → Controller接收参数 → Service校验:
    ① 检查phone格式(11位数字)
    ② 根据phone查询User表
    ③ BCryptUtil.matches()比对密码
    ④ 生成JWT Token(JwtUtil.generateToken())
    ⑤ Token存入Redis(设置24小时过期)
    ⑥ 返回Token + 用户基本信息

异常处理:
    - 手机号未注册 → 返回404 "用户不存在"
    - 密码错误 → 返回401 "密码错误"
    - 账户被锁定 → 返回403 "账户已被锁定"
```

#### U02 用户注册 - 核心流程
```
前端请求 → Controller接收参数 → Service处理:
    ① 校验验证码是否正确(Redis中取)
    ② 检查phone是否已注册
    ③ BCryptUtil.encrypt()加密密码
    ④ 创建User对象并插入数据库
    ⑤ 自动生成JWT Token(注册即登录)
    ⑥ 返回UserId + Token

数据校验规则:
    - 手机号: 正则^1[3-9]\d{9}$
    - 密码: 8-20位，必须含字母和数字
    - 昵称: 2-20位，不能含特殊字符
```

**【建这些API有什么用】**
- **U01登录**: 整个系统的入口，所有后续请求都依赖Token
- **U02注册**: 新用户入驻的唯一途径
- **U03获取信息**: 个人中心页面展示用户头像、昵称等
- **U04更新信息**: 用户修改个人资料
- **U05改密码**: 安全功能，定期更换密码
- **U06退出登录**: 安全退出，清除服务端会话

SELECT 
    record_time,
    value,
    AVG(value) OVER (
        ORDER BY record_time 
        ROWS BETWEEN 6 PRECEDING AND CURRENT ROW
    ) as ma_7day,
    -- 判断是否在正常范围
    CASE 
        WHEN dataType='blood_pressure' 
             AND CAST(SUBSTRING_INDEX(value,'/',1) AS DECIMAL)>140 
        THEN 'high'
        ELSE 'normal'
    END as status
FROM health_records 
WHERE user_id = #{userId} 
  AND data_type = #{dataType}
  AND record_time >= DATE_SUB(NOW(), INTERVAL #{days} DAY)
ORDER BY record_time ASC;

**【AI01对话接口 - 核心技术实现】**
```java
@RestController
@RequestMapping("/api/ai")
public class AiChatController {
    
    @Autowired
    private DeepSeekService deepSeekService;
    
    @Autowired
    private SiliconFlowService siliconFlowService;
    
    @PostMapping("/chat")
    public SseEmitter chat(@RequestBody AiChatRequestDTO request) {
        // 1. 创建SSE连接(流式输出)
        SseEmitter emitter = new SseEmitter(30000L); // 30秒超时
        
        // 2. 异步调用AI服务
        CompletableFuture.runAsync(() -> {
            try {
                // 3. 构建System Prompt(根据mode切换)
                String systemPrompt = buildSystemPrompt(request.getMode());
                
                // 4. 获取对话历史(最近10轮)
                List<Message> history = getConversationHistory(request.getConversationId());
                
                // 5. 调用DeepSeek API(流式)
                deepSeekService.chatStream(
                    systemPrompt, 
                    request.getMessage(), 
                    history,
                    chunk -> emitter.send(SseEmitter.event().data(chunk))
                );
                
                // 6. 发送完成信号
                emitter.complete();
            } catch (Exception e) {
                emitter.completeWithError(e);
            }
        });
        
        return emitter;
    }
}
```

### 8.3 常见问题FAQ

**Q1: 项目无法启动怎么办？**
A: 检查以下几点：
1. JDK版本是否为17+
2. MySQL是否启动并可连接
3. Redis是否启动
4. application.yml配置是否正确
5. 查看startup日志中的错误信息

**Q2: API接口返回404怎么办？**
A: 检查以下几点：
1. Controller类是否有@RestController注解
2. 请求路径是否正确（区分大小写）
3. 请求方法是否匹配（GET/POST/PUT/DELETE）
4. 应用端口是否正确（默认8080）

**Q3: 数据库连接失败怎么办？**
A: 检查以下几点：
1. MySQL服务是否启动
2. 数据库是否存在：CREATE DATABASE IF NOT EXISTS medical_health
3. 用户名密码是否正确
4. 是否有远程连接权限
5. 防火墙是否开放3306端口

**Q4: Redis连接失败怎么办？**
A: 检查以下几点：
1. Redis服务是否启动：redis-server
2. Redis是否监听默认端口6379
3. redis-cli ping是否能返回PONG
4. application.yml中redis配置是否正确

**Q5: JWT Token过期怎么办？**
A: 
1. 默认Token有效期为24小时
2. 前端应在Token过期前自动刷新
3. 如需调整有效期，修改JwtUtil.java中的expiration属性
4. 生产环境建议使用Redis存储Token黑名单

**Q6: 如何进行性能优化？**
A: 从以下几个方面入手：
1. 数据库：添加索引、优化SQL、使用连接池
2. 缓存：热点数据放入Redis、合理设置过期时间
3. JVM：调整堆内存大小、选择合适的垃圾回收器
4. 代码：避免N+1查询、使用异步处理、减少锁竞争
5. 架构：考虑引入消息队列、读写分离、分库分表
