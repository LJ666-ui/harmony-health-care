# 手动修复步骤（推荐）

由于批处理脚本可能出现编码问题，建议使用以下手动修复方法：

## 方法1: 使用MySQL命令行

### 步骤1: 打开MySQL命令行
```bash
mysql -u root -p
```
输入您的MySQL root密码

### 步骤2: 选择数据库
```sql
USE medical_health;
```

### 步骤3: 修复医生账号状态
```sql
UPDATE doctor SET status = 1 WHERE id = 1;
```

### 步骤4: 修复管理员密码
```sql
UPDATE admin SET password = '$2a$10$drI53FwlB2.tdaOYC4MkCOKlAW4PiYG0qLn.S7LoiJq07QcJpZEV.' WHERE username = 'admin';
```

### 步骤5: 验证修复结果
```sql
-- 检查医生状态
SELECT id, phone, status FROM doctor WHERE id = 1;

-- 检查管理员密码
SELECT id, username, LEFT(password, 30) AS password_prefix, status FROM admin WHERE username = 'admin';
```

### 步骤6: 退出MySQL
```sql
EXIT;
```

## 常见问题

### Q1: 找不到数据库 medical_health
**解决**: 
```sql
SHOW DATABASES;
```
查看数据库名称是否正确，可能名称不同

### Q2: UPDATE语句执行失败
**解决**: 
- 检查是否有足够的权限
- 确认表名和字段名是否正确
- 使用 `DESCRIBE doctor;` 和 `DESCRIBE admin;` 查看表结构

### Q3: 修复后仍无法登录
**解决**:
1. 重启后端服务
2. 清除浏览器缓存
3. 检查后端日志确认具体错误
4. 确认前端API配置正确

## 技术说明

### 为什么修改这些字段？

1. **doctor.status = 1**
   - 代码检查: `if (doctor.getStatus() != 1)` 返回"账号已被禁用"
   - status=0表示禁用，status=1表示启用

2. **admin.password**
   - 使用BCrypt加密验证
   - 原hash不正确，替换为已知正确的hash
   - 新hash对应密码: `123456`

### BCrypt密码说明
- Hash格式: `$2a$10$...` (rounds=10)
- 验证方式: `BCryptUtil.matches(rawPassword, encodedPassword)`
- 相同密码每次生成的hash不同（包含随机salt）
- 但验证时可以正确匹配
