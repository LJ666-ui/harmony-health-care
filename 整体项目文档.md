## 📌 一、这是什么项目？

### 一句话介绍

**星云医疗助手**是一款专为老年人设计的智慧健康管理平台，基于鸿蒙系统（HarmonyOS NEXT）开发，集成AI大模型（DeepSeek），提供健康数据记录、共病风险评估、智能康复指导、多设备协同等一站式服务。

### 项目背景

**为什么做这个？**

中国老龄化问题日益严重：
- 60岁以上人口超2.8亿（占总人口20%以上）
- 75%以上的老年人患有至少1种慢性病
- 医疗资源分布不均，基层医疗服务能力不足
- 老年人数字鸿沟严重，现有健康管理APP操作复杂

**我们想解决什么问题？**

1. **健康数据分散** → 统一记录血压/血糖/心率等，自动生成趋势图
2. **风险发现滞后** → AI算法自动评估5种疾病风险，早发现早干预
3. **专业指导缺乏** → DeepSeek AI提供24小时个性化健康咨询
4. **康复训练难坚持** → 3D动作演示+进度跟踪+智能推荐
5. **多设备不互通** → 利用鸿蒙分布式能力，手机/手表/平板数据同步
6. **适老化体验差** → 大字体大按钮高对比度语音控制全方位适配

### 目标用户

**主要用户**：60-80岁的老年人群
- 可能患有高血压、糖尿病等慢性病
- 希望在家就能管理自己的健康状况
- 不太擅长使用复杂的智能手机APP
- 子女不在身边，需要自我健康管理工具

**次要用户**：
- 患者家属（查看家人健康数据）
- 基层医生（管理患者病历、转院协调）
- 社区健康管理人员

### 产品定位

**不是什么**：
- ❌ 不是医院HIS系统（不做挂号缴费）
- ❌ 不是在线问诊平台（不开处方药）
- ❌ 不是可穿戴设备厂商（不生产硬件）

**是什么**：
- ✅ 是老年人的"数字健康管家"
- ✅ 是家庭医生的"辅助诊断工具"
- ✅ 是子女了解父母健康的"远程窗口"
- ✅ 是社区养老的"数字化解决方案"

##### **common/utils/** - 工具函数库

| 文件 | 功能 | 核心能力 |

| [HttpUtil.ets](entry/src/main/ets/common/utils/HttpUtil.ets) | HTTP请求封装 | GET/POST/PUT/DELETE<br>自动携带JWT Token<br>请求/响应拦截器<br>统一错误处理 |

#### **5️⃣ store/** - 全局状态管理

| 文件 | 功能 | 存储内容 |
|------|------|---------|
| [AppStore.ets](entry/src/main/ets/store/AppStore.ets) | 应用状态中心 | 当前登录用户<br>JWT Token<br>适老化模式开关<br>全局设置项 |

**工作原理**：
类似Vue的Vuex或React的Redux：
1. 登录成功 → 把用户信息和Token存入AppStore
2. 其他页面 → 直接从AppStore读取，无需再次请求后端
3. 退出登录 → 清空AppStore，跳转登录页

**优势**：
- 减少不必要的HTTP请求
- 跨页面共享数据（无需通过路由传参）
- 响应式更新（数据变化自动刷新UI）

### 🔧 后端详细结构（Spring Boot服务）

#### **核心定位**：
运行在服务器上的**API服务程序**，负责处理业务逻辑、访问数据库、提供数据接口。

#### **1️⃣ controller/** - 控制器层（8个控制器）

**是什么？**
后端的"前台接待员"，接收前端发来的HTTP请求。

**有什么用？**
- 定义RESTful API路径（如 `/api/user/login`）
- 接收和校验请求参数
- 调用Service层处理业务
- 返回JSON格式的响应结果

**实现了什么功能？**

| 控制器 | API路径前缀 | 负责模块 | 接口数量 | 核心功能 |
|--------|------------|---------|---------|---------|

| [HealthRecordController](src/main/java/com/example/medical/controller/HealthRecordController.java) | `/health-record` | 健康记录 | 7个 | CRUD操作/批量导入/按类型统计/趋势查询 |
| [HospitalController](src/main/java/com/example/medical/controller/HospitalController.java) | `/hospital` | 医院服务 | 6个 | 搜索/筛选/距离计算/详情查看 |

**距离计算公式（Haversine）**：
```java
public double calculateDistance(double lat1, double lon1, 
                               double lat2, double lon2) {
    double R = 6371; // 地球半径（公里）
    double dLat = Math.toRadians(lat2 - lat1);
    double dLon = Math.toRadians(lon2 - lon1);
    double a = Math.sin(dLat/2) * Math.sin(dLat/2) +
               Math.cos(Math.toRadians(lat1)) * Math.cos(Math.toRadians(lat2)) *
               Math.sin(dLon/2) * Math.sin(dLon/2);
    double c = 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1-a));
    return R * c; // 返回公里数
}
```

### 模块三：共病风险评估（⭐⭐⭐ 最大亮点）

**做什么**：根据健康数据自动分析多种疾病风险等级 ⭐ 这是项目的核心竞争力！

**为什么叫"共病"？**
因为很多老年人同时患多种疾病（高血压+糖尿病+骨质疏松...），单一疾病评估不够全面。我们的系统能同时评估5种常见老年疾病风险。

**评估的5项风险**：

| # | 风险名称 | 评估依据 | 风险因素 |
|---|---------|---------|---------|
| 1 | 🩺 高血压风险 | 收缩压/舒张压/年龄/吸烟史 | ≥140/90或年龄>65 |
| 2 | 💉 糖尿病风险 | 空腹血糖/餐后血糖/BMI/家族史 | >7.0或BMI>25 |
| 3 | 🧓 跌倒风险 | 年龄/肌力/平衡能力/用药情况 | >75岁或用降压药 |
| 4 | 💪 衰弱风险 | 体重下降/疲劳感/握力/步速 | 近期体重下降>5% |
| 5 | 🏋️ 肌少症风险 | 年龄/性别/活动量/营养状况 | 缺乏运动+营养不良 |

**评估流程**：

```
开始评估
    ↓
Step 1: 基本信息（年龄/性别/身高/体重→自动算BMI）
    ↓
Step 2: 血压数据（收缩压/舒张压）
    ↓
Step 3: 血糖数据（空腹/餐后）
    ↓
Step 4: 生活习惯（吸烟/饮酒/运动频率）
    ↓
Step 5: 既往病史（多选框勾选）
    ↓
提交（显示"正在分析..."动画3-5秒）
    ↓
结果页展示：
┌─────────────────────────────┐
│     ⭕ 综合风险等级          │
│     🔴 中风险               │
│   "需要注意以下几点"        │
├─────────────────────────────┤
│ 五项风险卡片（横向滑动）     │
│ [高血压🟡中] [糖尿病🟢低]   │
│ [跌倒🟡中] [衰弱🟢低]       │
│ [肌少症🟢低]                │
├─────────────────────────────┤
│ AI解读建议（Markdown渲染）   │
│                             │
│ 张大爷您好！                 │
│ 根据您的数据分析：           │
│ 1. 血压145略偏高...         │
│ 2. 建议...                  │
│ 💪 加油！                   │
│ *仅供参考，不适请就医*       │
├─────────────────────────────┤
│ [详细报告] [分享] [重新评估] │
└─────────────────────────────┘
```

**AI建议生成**：
- 调用DeepSeek API
- Prompt模板根据风险等级调整语气
- 低风险："您的基础状况不错，继续保持！"
- 中风险："有几点需要注意，但不用太担心"
- 高风险："建议尽快就医检查，同时..."
- 所有回答末尾加免责声明

**涉及接口**：3个
**涉及页面**：2个（表单/结果）

### 模块四：AI智能健康顾问（⭐⭐⭐ 第二大亮点）

**做什么**：让老人用自然语言问任何健康问题，获得专业又通俗的回答

**使用场景举例**：

| 用户提问 | AI回答要点 |
|---------|----------|
| "我今年65岁，血压有点高怎么办？" | 饮食建议+运动方案+监测频率+何时就医 |
| "糖尿病患者平时吃什么好？" | 低GI食物推荐+三餐示例+禁忌食物 |
| "老年人如何预防跌倒？" | 居家环境改造+平衡力训练+用药注意 |
| "高血压患者能运动吗？" | 推荐运动类型+强度+注意事项 |
| "我最近总是很累是怎么回事？" | 可能原因排查+自查项目+建议检查 |

**对话界面特点**：
- 聊天式交互（像微信一样直观）
- 消息气泡区分（用户蓝色/AI灰色）
- Markdown渲染（粗体/列表/链接）
- 加载动画（"AI正在思考..."）
- 快捷问题按钮（不知道问什么？点这里）
- 上下文连续（提到"刚才说的血压"AI能理解）
- 免责声明（每条回复底部显示）

**技术实现**：

## 🎨 八、设计规范与适老化标准

### 适老化设计原则（核心！）

**为什么重要？**
这是给老年人用的APP，如果年轻人觉得好用但老年人不会用，那就是失败的！

#### 1. 字体规范

| 场景 | 最小字号 | 推荐字号 | 老年模式放大 |
|-----|---------|---------|------------|
| 页面标题 | 20px | 24px | +20%（29px）|
| 正文内容 | 16px | 18px | +30%（23px）|
| 辅助文字 | 14px | 14px | +50%（21px）|
| 按钮文字 | 16px | 18px | +30%（23px）|

**4档切换**：小/中/大/特大（默认"中"，老年模式自动切到"大"）

#### 2. 按钮尺寸

| 场景 | 最小尺寸 | 老年模式 | 说明 |
|-----|---------|---------|------|
| 普通按钮 | 44×44px | 56×56px | Apple HIG标准 |
| 重要按钮（登录/提交） | 48×48px | 64×64px | 更醒目 |
| 图标按钮 | 40×40px | 48×48px | 导航栏图标 |
| Tab切换 | 44×44px高度 | 56×56px | 底部导航栏 |

#### 3. 颜色对比度

**遵循WCAG AA标准**：文字与背景对比度 ≥ 4.5:1

| 元素 | 正常模式 | 老年模式 |
|-----|---------|---------|
| 主文字 | #333 on #FFF（12.6:1）✅ | #000 on #FFF（21:1）✅ |
| 辅助文字 | #666 on #FFF（5.7:1）✅ | #333 on #FFF（12.6:1）✅ |
| 禁用文字 | #999 on #FFF（2.9:1）❌ | #666 on #FFF（5.7:1）✅ |
| 链接文字 | #007AFF on #FFF（4.5:1）✅ | #0055AA on #FFF（6.5:1）✅ |

**状态颜色**（全球统一）：
- 🟢 成功/正常/低风险：#52C41A（绿色）
- 🟡 警告/中风险：#FAAD14（黄色）
- 🔴 错误/高风险：#F5222D（红色）
- 🔵 信息/链接：#1890FF（蓝色）
- ⚪ 禁用/空状态：#D9D9D9（灰色）

#### 4. 间距规范

| 场景 | 最小值 | 推荐值 | 老年模式 |
|-----|-------|-------|---------|
| 页面边距 | 16px | 20px | 24px |
| 卡片内边距 | 12px | 16px | 20px |
| 元素间距 | 8px | 12px | 16px |
| 行高 | 1.5倍 | 1.6倍 | 1.8倍 |

#### 5. 交互动效

**老年模式原则**：减少动画、降低复杂度、增加反馈明确性

| 动效类型 | 正常模式 | 老年模式 |
|---------|---------|---------|
| 页面转场 | 300ms滑动 | 200ms淡入淡出 |
| 按钮点击 | 100ms缩放 | 150ms颜色变深 |
| 列项出现 | 依次渐显 | 直接显示 |
| Loading | 旋转动画 | 文字"加载中..." |
| 下拉刷新 | 弹簧回弹 | 简单箭头旋转 |

#### 6. 特殊适老化功能

- **语音播报**：重要信息TTS朗读（如评估结果、异常警告）
- **大字模式一键切换**：设置页顶部醒目入口
- **高对比模式**：黑白高对比配色方案
- **简化模式**：隐藏高级功能只保留核心操作
- **紧急呼叫**：首页固定悬浮按钮（预留SOS功能）
- **家属通知**：异常数据自动推送短信/微信（预留）

## 🔟 九、六大创新亮点（答辩重点）

### 创新点1：⭐ 共病风险评估算法

**是什么**：
首创针对老年人的"五维共病风险评估模型"，一次性评估高血压/糖尿病/跌倒/衰弱/肌少症五种常见老年疾病风险。

**技术亮点**：
- 多维度数据融合（生理指标+生活习惯+病史）
- 加权评分算法（不同因素权重不同）
- AI生成个性化建议（DeepSeek API）
- 可视化结果展示（颜色+图标+大字）

**竞品对比**：
- 大多数健康APP只评估单一疾病
- 我们同时评估5种且有联动效应
- 算法透明可解释（告诉用户为什么是这个风险等级）

**价值**：
- 早发现早干预（降低并发症风险）
- 减少重复检查（一次评估覆盖多个维度）
- 个性化指导（不是通用建议而是针对性方案）

---

### 创新点2：⭐ DeepSeek AI深度融合

**是什么**：
将国产大语言模型DeepSeek深度集成到健康咨询场景，提供24小时专业又通俗的健康问答服务。

**技术亮点**：
- Prompt工程（针对老年用户优化语气和内容长度）
- 上下文感知（结合用户健康档案回答）
- 多轮对话（保持上下文连贯）
- 安全护栏（免责声明+敏感词过滤+降级方案）

**应用场景**：
- 日常健康咨询（"血压高怎么办？"）
- 风险评估报告解读（"这个结果意味着什么？"）
- 康复训练指导（"这个动作怎么做才标准？"）
- 用药疑问解答（"这两种药能一起吃吗？"）

**价值**：
- 降低医疗咨询门槛（不用去医院就能问）
- 缓解医疗资源紧张（AI分担简单问题）
- 提升用户粘性（每天都能用的新功能）

---

### 创新点3：⭐ 3D康复训练演示

**是什么**：
利用Web/3D技术展示康复训练的标准动作，用户可以从任意角度观察学习。

**技术亮点**：
- 3D模型加载（.glb/.gltf格式）
- 手势交互（旋转/缩放/平移）
- 视频辅助（3D+视频双重展示）
- 进度跟踪（完成打勾+百分比）

**实现方案**：
- 方案A：ArkUI原生3D组件（如果有）
- 方案B：Web组件加载Three.js/WebGL
- 方案C：图片序列帧动画（降级方案）

**价值**：
- 比传统2D图片/文字指导更直观
- 可以360度观察细节（手肘弯曲角度等）
- 提升训练正确性和安全性

---

### 创新点4：⭐ 鸿蒙分布式协同

**是什么**：
利用鸿蒙系统的分布式软总线能力，实现手机/手表/平板多设备间数据无缝同步和协同。

**技术亮点**：
- 分布式设备发现（自动扫描附近鸿蒙设备）
- 分布式数据对象（数据跨设备实时同步）
- 分布式任务迁移（手机上的任务可以流转到平板）
- 超级终端体验（多设备融为一体）

**典型场景**：
- 手机测血压 → 手表同步显示
- 平板看康复视频 → 手机更新进度
- 手表检测异常 → 手机推送预警
- 车机导航 → 手机推送附近医院

**价值**：
- 充分发挥鸿蒙生态优势（差异化竞争力）
- 提升用户体验（数据不孤立在单一设备）
- 符合未来趋势（万物互联）

---

### 创新点5：⭐ 全方位适老化设计

**是什么**：
不是简单地把字体放大，而是从视觉/交互/内容/功能四个维度系统性适配老年人需求。

**设计理念**：
- **视觉适老**：大字体/高对比/清晰图标/舒适配色
- **交互适老**：大按钮/简化流程/容错性强/明确反馈
- **内容适老**：通俗语言/分条列出/重点突出/温暖语气
- **功能适老**：语音控制/紧急呼叫/家属通知/简化模式

**具体体现**：
- 4档字体大小切换（小/中/大/特大）
- 所有按钮最小44×44px（老年模式56×56px）
- 语音助手替代打字（"帮我记一下血压12582"）
- AI回答用大白话不说术语（"收缩压就是高压"）
- 异常数据主动提醒（"您的血压偏高，请注意！"）

**价值**：
- 真正解决老年人数字鸿沟问题
- 提升产品口碑（"这是我用过最简单的健康APP"）
- 扩大用户群体（不仅老年人，视障人士也受益）

### 应急预案详细说明

**预案A：DeepSeek API完全不可用**

触发条件：连续3次调用失败或返回错误

降级方案：
1. 使用本地规则引擎生成简单建议（if-else规则）
2. 准备100条常见问题的预设回答（静态JSON）
3. UI显示"AI正在学习中，以下是基于医学指南的建议"
4. 不影响核心流程（评估算法是本地的，不依赖API）

恢复条件：API恢复正常后自动切回

**预案B：3D模型无法加载**

触发条件：Web组件报错或模型文件损坏

降级方案：
1. 展示该动作的高清视频（必须有）
2. 展示2D分解图（步骤截图）
3. 展示文字要领说明（必须有）
4. 隐藏3D区域或显示"3D演示开发中"

**预案C：演示现场某个功能崩溃**

触发条件：点击某个按钮APP闪退或白屏

应对措施：
1. 立即跳过该环节，演示下一个功能
2. 使用预录制的视频/GIF片段代替实时演示
3. 口头说明："这个功能在现场环境中遇到了临时问题，我们在开发环境中已经验证过，现在为您播放录屏"
4. 不要在现场尝试修复（浪费时间且显得不专业）

**预案D：时间严重不够（只剩10天但只完成了50%）**

触发条件：Day 11评估发现进度严重滞后

应对措施：
1. 召开紧急会议重新排优先级
2. 只保留P0功能（登录/首页/记录/评估/AI）
3. 砍掉所有P2功能（设备/古医图/统计排行）
4. P1功能保留骨架（UI有了但可能没有真实数据）
5. 集中所有精力确保核心流程跑通

**HttpUtil.ets 核心实现**：
```typescript
// 自动Token管理
async request<T>(config: RequestConfig): Promise<Result<T>> {
  // 1. 从Storage读取Token
  const token = StorageUtil.get('token')
  
  // 2. 附加到请求头
  config.headers = { ...config.headers, 'Authorization': `Bearer ${token}` }
  
  // 3. 发送请求（超时10秒）
  const response = await fetch(config.url, {
    method: config.method,
    headers: config.headers,
    body: JSON.stringify(config.data)
  })
  
  // 4. 解析响应
  const result: Result<T> = await response.json()
  
  // 5. 错误码处理
  if (result.code === 401) {
    // Token过期 → 清除 → 跳转登录
    StorageUtil.remove('token')
    router.pushUrl({ url: 'pages/Login' })
  }
  
  return result
}
```

**RiskAssessmentService（风险评估算法）- 核心亮点**：
```java
@Service
public class RiskAssessmentServiceImpl implements RiskAssessmentService {
    
    /**
     * 执行完整的五维共病风险评估
     * 输入：用户基本信息 + 生活习惯 + 家族病史 + 体检数据
     * 输出：五项风险等级 + AI建议
     */
    public RiskResult assessAllRisks(Long userId, RiskAssessmentDTO dto) {
        RiskResult result = new RiskResult();
        
        // 1. 高血压风险评估（基于年龄/BMI/家族史/吸烟/血压值）
        double bpScore = calculateHypertensionRisk(dto);
        result.setHypertensionRisk(mapToLevel(bpScore));
        
        // 2. 糖尿病风险评估（基于年龄/BMI/家族史/血糖值/运动习惯）
        double diabetesScore = calculateDiabetesRisk(dto);
        result.setDiabetesRisk(mapToLevel(diabetesScore));
        
        // 3. 心血管疾病风险评估（基于血压/血脂/吸烟/年龄/性别）
        double cvdScore = calculateCardiovascularRisk(dto);
        result.setCardiovascularRisk(mapToLevel(cvdScore));
        
        // 4. 跌倒风险评估（基于年龄/平衡能力/用药情况/视力）
        double fallScore = calculateFallRisk(dto);
        result.setFallRisk(mapToLevel(fallScore));
        
        // 5. 衰弱/肌少症风险评估（基于体重下降/握力/步速）
        double frailtyScore = calculateFrailtyRisk(dto);
        result.setFrailtyRisk(mapToLevel(frailtyScore));
        
        // 6. 计算综合健康评分（0-100）
        result.setOverallScore(calculateOverallScore(result));
        
        // 7. 生成AI建议（调用DeepSeek API）
        String aiSuggestion = aiService.generateHealthSuggestion(result, dto);
        result.setAiSuggestion(aiSuggestion);
        
        // 8. 保存到数据库
        saveAssessmentResult(userId, result);
        
        return result;
    }
    
    /**
     * 高血压风险计算算法（简化版）
     * 考虑因素：年龄(+) BMI(+) 家族史(++) 吸烟(++) 当前血压值(+++)
     */
    private double calculateHypertensionRisk(RiskAssessmentDTO dto) {
        double score = 0;
        
        // 年龄因子（每增加10岁+10分）
        score += Math.max(0, (dto.getAge() - 40)) * 1.0;
        
        // BMI因子（BMI≥25开始加分）
        if (dto.getBmi() >= 25) score += 15;
        if (dto.getBmi() >= 30) score += 10;  // 肥胖额外加
        
        // 家族史（一级亲属患病+20分）
        if (dto.hasFamilyHistory("hypertension")) score += 20;
        
        // 吸烟（当前吸烟+15分）
        if ("current".equals(dto.getSmokingStatus())) score += 15;
        
        // 当前血压值（最关键因子）
        if (dto.getSystolicBp() >= 140 || dto.getDiastolicBp() >= 90) {
            score += 25;  // 已达高血压标准
        } else if (dto.getSystolicBp() >= 130 || dto.getDiastolicBp() >= 80) {
            score += 10;  // 高血压前期
        }
        
        // 归一化到0-100
        return Math.min(100, Math.max(0, score));
    }
    
    /**
     * 分数→等级映射
     * 0-30: 低风险（绿色）
     * 31-50: 中风险（黄色）
     * 51-70: 高风险（橙色）
     * 71-100: 极高风险（红色）
     */
    private String mapToLevel(double score) {
        if (score <= 30) return "low";
        if (score <= 50) return "medium";
        if (score <= 70) return "high";
        return "very_high";
    }
}
```

**AiService（AI对话服务）- 创新亮点**：
```java
@Service
public class AiServiceImpl implements AiService {
    
    @Value("${deepseek.api-key}")
    private String deepseekApiKey;
    
    @Value("${deepseek.api-url}")
    private String deepseekApiUrl;
    
    private final RestTemplate restTemplate;
    
    /**
     * 与DeepSeek AI进行对话
     * 支持上下文连续对话（传入conversationId）
     */
    public ChatResponse chatWithAI(String message, Long userId, String conversationId) {
        // 1. 构建系统提示词（角色设定+专业要求+适老化）
        String systemPrompt = buildSystemPrompt();
        
        // 2. 获取用户健康档案作为上下文
        String healthContext = buildHealthContext(userId);
        
        // 3. 构建请求体
        Map<String, Object> requestBody = new HashMap<>();
        requestBody.put("model", "deepseek-chat");
        requestBody.put("messages", Arrays.asList(
            Map.of("role", "system", "content", systemPrompt),
            Map.of("role", "user", "content", healthContext + "\n\n用户问题：" + message)
        ));
        requestBody.put("temperature", 0.7);  // 平衡创造性和准确性
        requestBody.put("max_tokens", 1000);  // 限制回复长度
        
        // 4. 调用DeepSeek API
        HttpHeaders headers = new HttpHeaders();
        headers.setContentType(MediaType.APPLICATION_JSON);
        headers.setBearerAuth(deepseekApiKey);
        
        HttpEntity<Map<String, Object>> entity = new HttpEntity<>(requestBody, headers);
        
        ResponseEntity<String> response = restTemplate.postForEntity(
            deepseekApiUrl, entity, String.class
        );
        
        // 5. 解析AI回复
        JSONObject jsonResponse = JSON.parseObject(response.getBody());
        String aiReply = jsonResponse.getJSONArray("choices")
            .getJSONObject(0)
            .getJSONObject("message")
            .getString("content");
        
        // 6. 保存对话历史到数据库
        saveChatHistory(userId, message, aiReply, conversationId);
        
        // 7. 返回结果
        return new ChatResponse(aiReply, generateConversationId());

@Component
public class JwtUtil {
    
    @Value("${jwt.secret}")
    private String secret;  // 密钥（至少256位随机字符串）
    
    @Value("${jwt.expiration}")
    private long expiration;  // 过期时间（毫秒，默认24小时=86400000）
    
    /**
     * 生成Token
     * Payload包含：userId + username + role + iat（签发时间）
     */
    public String generateToken(Long userId, String username, String role) {
        Date now = new Date();
        Date expiryDate = new Date(now.getTime() + expiration);
        
        return Jwts.builder()
                .setSubject(String.valueOf(userId))
                .claim("username", username)
                .claim("role", role)
                .setIssuedAt(now)
                .setExpiration(expiryDate)
                .signWith(SignatureAlgorithm.HS512, secret)
                .compact();
    }
    
    /**
     * 从Token中解析userId
     */
    public Long getUserIdFromToken(String token) {
        Claims claims = Jwts.parser()
                .setSigningKey(secret)
                .parseClaimsJws(token)
                .getBody();
        return Long.parseLong(claims.getSubject());
    }
    
    /**
     * 验证Token是否有效（未过期+签名正确）
     */
    public boolean validateToken(String token) {
        try {
            Jwts.parser().setSigningKey(secret).parseClaimsJws(token);
            return true;
        } catch (Exception e) {
            return false;  // 过期/签名无效/格式错误
        }
    }
}
```

##### **核心表详解（按重要性排序）**

**🔴 最高频使用（核心业务）**：

1. **user表（用户表）**
   - 存储所有用户信息（老人+医生+管理员）
   - 字段：id, phone, password(Bcrypt哈希), nickname, avatar, gender, birthday, role(user/doctor/admin), is_deleted
   - 索引：phone(UNIQUE), id(PK)
   - 测试数据：10条（5位老人+3位医生+2个管理员）

2. **health_record表（健康记录表）⭐最频繁**
   - 存储所有健康监测数据（血压/血糖/心率/体重/体温）
   - 字段：id, user_id(FK), type(ENUM), value(VARCHAR), unit, record_time, source(手动/设备), notes
   - 索引：(user_id, record_time)复合索引加速时间范围查询
   - 测试数据：200条（模拟30天连续监测）

3. **risk_assessment表（风险评估结果表）⭐核心算法产出**
   - 存储每次风险评估的五维结果
   - 字段：id, user_id, assess_time, hypertension_risk(JSON), diabetes_risk(JSON)... ai_suggestion(TEXT), overall_score
   - 测试数据：20条（不同用户的评估报告）

4. **chat_history表（AI对话历史表）⭐AI功能依赖**
   - 存储用户与AI的所有对话记录
   - 字段：id, user_id, role(user/assistant), content(TEXT), conversation_id, create_time
   - 索引：conversation_id（用于加载上下文）
   - 测试数据：100条（模拟50轮对话）

**🟡 高频使用（重要功能）**：

5. **hospital表（医院表）**
   - 381家医院数据（已导入）
   - 字段：id, name, level, address, latitude, longitude, phone, website, description
   - 索引：name(全文索引), level

6. **science_category表 + science_article表（科普文章）**
   - 50篇健康科普文章
   - 支持分类浏览/搜索/阅读量统计

7. **rehab_plan表 + rehab_action表 + rehab_progress表（康复训练）**
   - 康复计划/动作定义/训练进度
   - 支持3D模型URL和视频URL

**🟢 中低频使用（辅助功能）**：

8. **medical_record表（电子病历）**
9. **transfer_application表（转院申请）**
10. **device_binding表（设备绑定）**
11. **food表（食材百科，744种）**
