# 星云医疗助手 - 前端分阶段开发提示词

## 项目概述
- **项目名称**：星云医疗助手
- **技术栈**：HarmonyOS SDK 6.0.2 (API 22)、ArkTS、ArkUI 9.0.0
- **开发环境**：DevEco Studio 6.0.2、JDK 1.8.0_371、Node.js v20.19.6
- **项目目标**：构建一个分布式智慧医疗应用，提供健康管理、智能问诊、预约挂号等核心功能

## 阶段一：环境搭建与基础准备（第1-2天）

### 阶段目标
- 完成项目初始化与环境配置
- 搭建基础目录结构
- 实现公共组件与工具类

### 开发提示词

#### 1. 项目初始化
```
请在 DevEco Studio 6.0.2 中创建一个新的 HarmonyOS 项目，使用 API 22，选择 "Empty Ability" 模板。项目名称为 "NebulaMedicalAssistant"，包名为 "com.nebulamedical.assistant"。
```

#### 2. 目录结构搭建
```
请按照以下结构搭建项目目录：
- entry/src/main/ets/
  - pages/          # 页面目录
  - common/         # 公共资源
    - components/   # 公共组件
    - constants/    # 常量定义
    - utils/        # 工具类
    - styles/       # 样式定义
  - model/          # 数据模型
  - services/       # 服务层
  - App.ets         # 应用入口
```

#### 3. 公共组件开发
```
请创建以下公共组件：
- Header.ets：页面头部组件，包含标题和返回按钮
- Button.ets：自定义按钮组件，支持不同样式
- Card.ets：卡片组件，用于展示信息
- Loading.ets：加载状态组件
```

#### 4. 工具类封装
```
请封装以下工具类：
- StorageUtil.ets：本地存储工具，封装 Preferences 操作
- HttpUtil.ets：网络请求工具，支持 Mock 数据
- TimeUtil.ets：时间处理工具
- ValidatorUtil.ets：数据验证工具
```

#### 5. 样式定义
```
请在 common/styles/ 目录下创建样式文件：
- theme.ets：主题配置，包含颜色、字体等
- commonStyles.ets：通用样式，如间距、圆角等
```

### 无后端依赖任务
- ✅ 项目初始化与目录结构搭建
- ✅ 公共组件开发
- ✅ 工具类封装
- ✅ 样式定义

## 阶段二：核心页面开发（第3-5天）

### 阶段目标
- 实现登录页面
- 实现首页（Index）
- 实现主页（HomePage）
- 实现健康记录页面

### 开发提示词

#### 1. 登录页面（Login.ets）
```
请实现登录页面，包含以下功能：
- 用户名输入框（支持手机号/身份证号）
- 密码输入框（支持明文/密文切换）
- 登录按钮
- 忘记密码链接
- 注册账号链接
- 登录成功后跳转到 HomePage
```

#### 2. 首页（Index.ets）
```
请实现首页，包含以下功能：
- 应用 logo
- 应用名称 "星云医疗助手"
- 进入系统按钮，点击跳转到 Login 页面
- 背景图或背景色设置
```

#### 3. 主页（HomePage.ets）
```
请实现主页，包含以下功能：
- 顶部 Header，包含标题 "首页" 和返回按钮
- 功能模块入口：健康记录、智能问诊、预约挂号、用药提醒、健康报告、医院导航
- 快捷操作区：一键测血压、一键测血糖
- 健康数据概览：显示最近的健康指标
```

#### 4. 健康记录页面（HealthRecord.ets）
```
请实现健康记录页面，包含以下功能：
- 顶部 Header，包含标题 "健康记录" 和返回按钮
- 健康数据分类：血压、血糖、心率、体温
- 数据列表展示，显示时间和数值
- 添加数据按钮
- 数据趋势图表（使用 ArkUI 图表组件）
```

- ✅ 首页 UI 开发

## 技术要求与规范

### 1. 代码规范
- **类型安全**：使用显式类型，避免使用 any/unknown 类型
- **路由导入**：使用 `import { router } from '@kit.ArkUI'`
- **组件复用**：封装可复用组件，减少代码冗余
- **响应式布局**：使用 Flex、Grid 等布局方式，确保适配不同屏幕

### 2. 设计规范
- **颜色系统**：
  - 主色调：#3B82F6（蓝色）
  - 辅助色：#10B981（绿色）、#F59E0B（橙色）、#EF4444（红色）
  - 中性色：#F3F4F6（浅灰）、#9CA3AF（中灰）、#1F2937（深灰）
- **字体系统**：
  - 标题：20px，粗体
  - 副标题：16px，半粗体
  - 正文：14px，常规
  - 小字：12px，常规
- **间距系统**：4px、8px、12px、16px、24px、32px
- **圆角系统**：4px、8px、12px、16px、24px

### 3. 开发建议
- **模块化开发**：按功能模块划分代码，提高可维护性
- **组件化设计**：封装公共组件，减少代码冗余
- **增量开发**：逐步实现功能，确保每个阶段都有可交付成果
- **Mock 数据**：使用 Mock 数据，确保无后端时也能正常开发
- **版本控制**：使用 Git 进行版本控制，定期提交代码
