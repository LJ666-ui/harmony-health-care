## 一、项目概述

### 1.1 项目定位

**星云医疗助手**是一款基于 HarmonyOS 6.0.2 (API 22) 原生开发的分布式智慧医疗全周期管理应用，聚焦老年群体就医、康复、健康管理的核心痛点，打造"院前-院中-院后-日常科普"全场景闭环服务。

### 1.2 核心价值

- **提升就医效率**：全流程智能服务，减少等待时间
- **打破数据孤岛**：分布式数据协同，实现跨院数据共享
- **优化康复效果**：AI 共病评估 + 3D 康复指导
- **增强健康管理**：场景化科普学习，提升自我管理能力

### 1.3 目标用户

- **老年患者**：需要简单易用的医疗管理工具
- **家属**：远程关注患者健康状态
- **医护人员**：快速获取患者数据，AI 辅助诊断

## 二、技术要求

### 2.1 技术栈

- **核心框架**：HarmonyOS SDK 6.0.2 (API 22)
- **开发语言**：ArkTS
- **UI 框架**：ArkUI 9.0.0
- **路由**：@kit.ArkUI router
- **状态管理**：AppStore
- **本地存储**：SQLite 3.40.0、Preferences
- **端侧 AI**：MindSpore Lite 2.2.10
- **AR 功能**：AR Engine 11.0.0.300

### 2.2 开发环境

- **IDE**：DevEco Studio 6.0.2
- **JDK**：1.8.0_371
- **Node.js**：v20.19.6
- **Git**：2.52.0
- **ohpm**：0.10.0

### 2.3 代码规范

- **类型安全**：避免使用 `any` 类型，所有参数和返回值都要明确定义类型
- **路由导入**：使用 `import { router } from '@kit.ArkUI'` 而不是 `import router from '@ohos/router'`
- **组件复用**：优先使用公共组件，减少重复代码
- **响应式布局**：考虑不同设备的屏幕尺寸，使用自适应布局
- **用户体验**：添加加载状态、错误提示等交互反馈

### 3.2 第一周开发计划

**任务**：
- 搭建 HarmonyOS 开发环境
- 创建项目结构
- 开发基础组件（Header、Button、Card、Footer、Input）

**要求**：
- 组件必须可复用，支持自定义属性

- 组件要有良好的交互效果

#### 3.2.2 Day 3：首页与登录页面

**任务**：
- 开发 Index.ets（引导页）

**要求**：

- 登录页要有用户名、密码输入和登录按钮
- 路由跳转要使用 `@kit.ArkUI router`

**任务**：
- 开发 HomePage.ets（主页）
- 实现快捷功能入口
- 展示健康数据概览

**要求**：

- 快捷功能要链接到相应页面
- 健康数据要使用 Mock 数据展示

#### 3.2.4 Day 5：健康记录页面

**任务**：
- 开发 HealthRecords.ets（健康记录页面）
- 实现数据分类展示
- 添加健康数据记录功能

**要求**：
- 页面要有 Tab 选择器（全部、血压、血糖、心率、体重）
- 要展示健康数据列表
- 要有添加健康记录的功能

#### 3.2.5 Day 6-7：功能页面开发

**任务**：
- 开发 Appointments.ets（预约管理页面）
- 开发 Medications.ets（用药管理页面）
- 开发 Profile.ets（个人中心页面）
- 测试页面导航和功能

**要求**：
- 预约管理页面要展示预约列表和状态
- 用药管理页面要展示今日提醒和用药列表

## 四、设计规范

### 4.1 UI 设计

- **颜色系统**：
  - 主色：#1677FF
  - 背景色：#F5F7FA
  - 文字色：#333333（主）、#666666（次）、#999999（辅助）

- **字体系统**：
  - 标题：24px, 600 权重
  - 正文：16px, 400 权重
  - 按钮：18px, 600 权重

- **间距系统**：
  - 大间距：20px
  - 中等间距：16px
  - 小间距：8px

- **圆角系统**：
  - 按钮：24px
  - 卡片：12px
  - 输入框：8px

### 4.2 交互设计

- **按钮交互**：
  - 点击有反馈效果
  - 加载状态要显示
  - 禁用状态要明显

- **页面跳转**：
  - 使用 `router.pushUrl` 进行页面跳转
  - 使用 `router.back` 返回上一页
  - 登录后清空路由栈

- **数据展示**：
  - 使用卡片式布局
  - 重要数据要突出显示
  - 状态变化要有视觉反馈

### 4.3 响应式设计

- **布局适配**：
  - 使用 `Flex` 布局
  - 考虑不同屏幕尺寸
  - 确保在手机、平板上都能正常显示

- **字体适配**：
  - 使用相对单位
  - 确保字体在不同设备上清晰可读

### 8.2 问题解决
