# 登录问题修复指南

## 问题分析

### 问题1: 医生登录提示"账号已被禁用"
- **现象**: 使用手机号 `13812345001` 和密码 `123456` 登录医生端，提示"账号已被禁用"
- **原因**: 数据库 `doctor` 表中该医生的 `status` 字段为 `0`（禁用状态）
- **代码位置**: `DoctorController.java:68-70`
  ```java
  if (doctor.getStatus() == null || doctor.getStatus() != 1) {
      return Result.error("账号已被禁用");
  }
  ```

### 问题2: 管理员登录提示"密码错误"
- **现象**: 使用用户名 `admin` 和密码 `admin123` 登录管理后台，提示"密码错误"
- **原因**: 数据库 `admin` 表中的密码hash值不正确，无法通过BCrypt验证
- **代码位置**: `AdminController.java:66`
  ```java
  if (!BCryptUtil.matches(password, admin.getPassword())) {
      return Result.error("密码错误");
  }
  ```

## 解决方案

### 方案1: 执行SQL修复脚本（推荐）

1. **找到SQL修复脚本**
   - 文件位置: `fix_login_issues.sql`

2. **执行SQL脚本**
   ```bash
   # 方式1: 使用MySQL命令行
   mysql -u root -p medical_health < fix_login_issues.sql
   
   # 方式2: 使用MySQL Workbench或其他数据库工具
   # 打开 fix_login_issues.sql 文件并执行
   ```

3. **验证修复结果**
   - 脚本执行后会自动显示验证信息
   - 确认 `doctor` 表的 `status` 字段已更新为 `1`
   - 确认 `admin` 表的 `password` 字段已更新

### 方案2: 手动修改数据库

#### 修复医生账号状态
```sql
UPDATE `doctor` SET `status` = 1 WHERE `id` = 1;
```

#### 修复管理员密码
```sql

```

## 技术细节

### BCrypt密码加密
- 系统使用BCrypt算法进行密码加密
- BCrypt hash格式: `$2a$10$...` (rounds=10)
- 验证方式: `BCryptUtil.matches(rawPassword, encodedPassword)`

### 医生登录流程
1. 通过手机号查找 `user` 表
2. 验证密码（支持BCrypt和明文）
3. 通过 `user_id` 查找 `doctor` 表
4. 检查 `doctor.status` 是否为 `1`
5. 生成JWT Token并返回

### 管理员登录流程
1. 通过用户名查找 `admin` 表
2. 检查 `admin.status` 是否为 `1`
3. 使用BCrypt验证密码
4. 生成JWT Token并返回

## 常见问题

### Q: 为什么管理员密码从 admin123 改为 123456？
A: 为了使用数据库中已验证可用的BCrypt hash，避免重新生成hash的复杂性。如果需要使用 admin123，可以运行 `PasswordHashGenerator.java` 生成正确的hash。

### Q: 修复后仍然无法登录怎么办？
A: 
1. 检查数据库连接是否正常
2. 确认SQL脚本执行成功
3. 查看后端日志确认具体错误
4. 检查前端是否正确调用登录API

### Q: 如何验证BCrypt hash是否正确？
A: 可以使用在线BCrypt验证工具，或运行以下代码：
```java
BCryptPasswordEncoder encoder = new BCryptPasswordEncoder();
boolean matches = encoder.matches("123456", "$2a$10$drI53FwlB2.tdaOYC4MkCOKlAW4PiYG0qLn.S7LoiJq07QcJpZEV.");
System.out.println(matches); // 应该输出 true
```
