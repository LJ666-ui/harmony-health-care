# 管理员登录密码修复指南

## 问题现象
使用账号 `admin` 和密码 `admin123` 登录管理后台时提示"密码错误"。

## 问题原因
数据库中admin表的password字段存储的bcrypt哈希值与明文密码"admin123"不匹配。

## 解决方案

### 方案1：使用修复工具（推荐）

1. **启动后端服务**
   ```bash
   mvn spring-boot:run
   ```
   
   或者在IDEA中运行 `Application.java`

2. **访问密码验证接口**
   在浏览器或Postman中访问：
   ```
   GET http://localhost:8080/admin/verify-password?username=admin&password=admin123
   ```
   
   这会返回：
   - 当前存储的hash
   - 输入密码是否匹配
   - 如果不匹配，会提供正确的hash

3. **自动修复密码**
   访问以下接口会自动将admin用户的密码更新为 `admin123`：
   ```
   GET http://localhost:8080/admin/fix-admin-password
   ```

4. **重新登录**
   - 用户名：`admin`
   - 密码：`admin123`

### 方案2：手动生成hash并更新

1. **生成正确的hash**
   启动后端后访问：
   ```
   GET http://localhost:8080/admin/generate-password?password=admin123
   ```
   
   会返回SQL语句：
   ```sql
   UPDATE admin SET password = '$2a$10$...' WHERE username = 'admin';
   ```

2. **在数据库中执行SQL**
   使用Navicat或其他数据库工具执行上面生成的SQL语句。

### 方案3：直接在数据库中修复

1. **查看当前admin用户数据**
   ```sql
   SELECT id, username, password, real_name, role, status 
   FROM admin 
   WHERE username = 'admin';
   ```

2. **检查password字段**
   - 如果password为空或不以 `$2a$10$` 开头，说明格式不正确
   - 需要生成新的bcrypt hash

3. **启动后端生成hash**
   后端启动时会自动初始化admin用户（如果已配置）

## 验证步骤

1. 访问管理后台登录页面
2. 输入：用户名 `admin`，密码 `admin123`
3. 点击登录
4. 应该成功跳转到管理后台首页

## 密码加密说明

- 本系统使用 **BCrypt** 加密算法
- BCrypt每次生成的hash都不同（包含随机salt）
- 但验证时都能正确匹配
- Hash格式：`$2a$10$...`（$2a$表示BCrypt算法，$10$表示cost factor）

## 添加的修复工具

已在 `AdminPasswordFixController.java` 中添加以下接口：

1. **GET /admin/generate-password** - 生成密码hash
2. **GET /admin/fix-admin-password** - 自动修复admin密码
3. **GET /admin/verify-password** - 验证密码是否正确

## 注意事项

- 修复工具接口仅用于开发环境，生产环境应移除或添加权限控制
- 密码修复后，原有登录的token将失效，需要重新登录
- 如果admin用户不存在，需要先创建admin用户

## 完整修复流程

1. 启动后端服务
2. 访问 `http://localhost:8080/admin/fix-admin-password`
3. 看到成功提示
4. 回到登录页面，使用 `admin/admin123` 登录
5. 登录成功！
