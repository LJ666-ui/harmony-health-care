# 管理员登录接口测试指南

## 问题分析

### 1. Postman请求错误
**错误信息**：`Error: Request path contains unescaped characters`
**原因**：URL中包含了中文字符"向上还原"
**解决**：清除URL中的中文字符，只保留 `http://localhost:8080/admin/login`

### 2. 后端服务状态
从日志可以看到：
- ✅ Spring Boot已启动
- ✅ Tomcat在8080端口运行
- ✅ DispatcherServlet已初始化
- ⚠️ HikariPool警告：线程饥饿或时钟跳跃（不影响功能）

## 正确的测试步骤

### 方法1：使用Postman

#### 步骤1：配置请求
```
方法：POST
URL：http://localhost:8080/admin/login
```

#### 步骤2：设置请求头
```
Content-Type: application/json
```

#### 步骤3：设置请求体（Body）
选择 `raw` 和 `JSON` 格式：
```json
{
  "username": "admin",
  "password": "admin123"
}
```

#### 步骤4：发送请求
点击 `Send` 按钮

#### 预期响应
**成功**：
```json
{
  "code": 200,
  "message": "成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiJ9...",
    "adminId": 1,
    "username": "admin",
    "realName": "超级管理员",
    "role": 1,
    "avatar": null
  }
}
```

**失败**：
```json
{
  "code": 500,
  "message": "用户名不存在" // 或其他错误信息
}
```

### 方法2：使用curl命令

```bash
curl -X POST http://localhost:8080/admin/login \
  -H "Content-Type: application/json" \
  -d "{\"username\":\"admin\",\"password\":\"admin123\"}"
```

### 方法3：使用PowerShell

```powershell
$headers = @{"Content-Type"="application/json"}
$body = '{"username":"admin","password":"admin123"}'
Invoke-RestMethod -Method Post -Uri "http://localhost:8080/admin/login" -Headers $headers -Body $body
```

## 数据库检查

### 检查管理员数据
```sql
-- 查看admin表数据
SELECT
    id,
    username,
    LEFT(password, 20) AS password_prefix,  -- 只显示密码前20个字符
    real_name,
    role,
    status,
    is_deleted
FROM admin
WHERE username = 'admin';
```

### 预期结果
| id | username | password_prefix       | real_name    | role | status | is_deleted |
|----|----------|----------------------|--------------|------|--------|------------|
| 1  | admin    | $2a$10$N.zmdr9k7u... | 超级管理员   | 1    | 1      | 0          |

**密码说明**：
- `$2a$10$` 开头表示BCrypt加密
- 完整密码长度约60个字符
- 明文密码 `admin123` 加密后存储

### 如果密码未加密或错误

#### 方案1：使用后端接口加密（推荐）
创建一个临时接口来生成加密密码：

```java
// 在UserController中添加临时方法
@GetMapping("/encode-password")
public Result<?> encodePassword(@RequestParam String password) {
    String encoded = BCryptUtil.encrypt(password);
    return Result.success(encoded);
}
```

访问：`http://localhost:8080/user/encode-password?password=admin123`
获取加密后的密码，然后更新数据库。

#### 方案2：直接更新数据库
```sql
-- admin123的BCrypt加密结果（每次加密都不同，但验证时都能通过）
UPDATE admin
SET password = '$2a$10$N.zmdr9k7uOCQv37Y7Cb./8iB6W9W9W9W9W9W9W9W9W9W9W9W9W9W9',
    status = 1,
    is_deleted = 0
WHERE username = 'admin';
```

## 常见错误及解决

### 错误1：用户名不存在
**原因**：数据库中没有username='admin'的记录
**解决**：
```sql
INSERT INTO admin (username, password, real_name, role, status, create_time, is_deleted)
VALUES ('admin', '$2a$10$N.zmdr9k7uOCQv37Y7Cb./8iB6W9W9W9W9W9W9W9W9W9W9W9W9W9W9', '超级管理员', 1, 1, NOW(), 0);
```

### 错误2：密码错误
**原因**：数据库密码未加密或加密方式不正确
**解决**：使用上述方案1或方案2更新密码

### 错误3：账户已被禁用
**原因**：status字段为0
**解决**：
```sql
UPDATE admin SET status = 1 WHERE username = 'admin';
```

### 错误4：网络错误
**原因**：后端服务未启动或端口被占用
**解决**：
1. 检查后端是否启动：访问 http://localhost:8080
2. 检查端口占用：`netstat -ano | findstr :8080`

## 密码加密验证

### 如何验证密码是否正确加密？

在数据库中查询：
```sql
SELECT username, password FROM admin WHERE username = 'admin';
```

**正确格式**：
```
$2a$10$N.zmdr9k7uOCQv37Y7Cb./8iB6W9W9W9W9W9W9W9W9W9W9W9W9W9W9
```

**特征**：
- 以 `$2a$` 开头（BCrypt算法标识）
- 总长度约60个字符
- 包含 `$` 分隔符

**错误格式**：
- `admin123`（明文）
- `e10adc3949ba59abbe56e057f20f883e`（MD5，不安全）
