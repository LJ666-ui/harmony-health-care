# 管理员登录问题诊断指南

## 问题现象
管理员登录时提示"登录已过期，请重新登录"

## 诊断步骤

### 1. 检查后端服务是否运行
```bash
# 检查8080端口是否被占用
netstat -ano | findstr :8080

# 或者在浏览器访问
http://localhost:8080/user/login
```

### 2. 检查数据库管理员数据
执行以下SQL查询：
```sql
-- 查看admin表数据
SELECT id, username, real_name, role, status, is_deleted FROM admin WHERE is_deleted = 0;

-- 查看admin表的密码（检查是否已加密）
SELECT id, username, password FROM admin WHERE username = 'admin';
```

### 3. 测试管理员登录接口
使用Postman或curl测试：
```bash
POST http://localhost:8080/admin/login
Content-Type: application/json

{
  "username": "admin",
  "password": "admin123"
}
```

### 4. 检查前端网络请求
打开浏览器开发者工具（F12）：
1. 切换到Network标签
2. 点击登录按钮
3. 查看是否有 `/admin/login` 请求
4. 检查请求状态码和响应内容

## 常见问题及解决方案

### 问题1：后端服务未启动
**现象**：Network请求失败，显示"网络错误"
**解决**：启动Spring Boot应用

### 问题2：数据库密码未加密
**现象**：登录失败，提示"密码错误"
**解决**：更新数据库密码为BCrypt加密后的值
```sql
-- 使用BCrypt加密密码
-- admin123 加密后：$2a$10$N.zmdr9k7uOCQv37Y7Cb./8iB6W9W9W9W9W9W9W9W9W9W9W9W9W9W9

UPDATE admin SET password = '$2a$10$N.zmdr9k7uOCQv37Y7Cb./8iB6W9W9W9W9W9W9W9W9W9W9W9W9W9W9' WHERE username = 'admin';
```

### 问题3：管理员账户被禁用
**现象**：登录失败，提示"账户已被禁用"
**解决**：
```sql
UPDATE admin SET status = 1 WHERE username = 'admin';
```

### 问题4：管理员数据不存在
**现象**：登录失败，提示"用户名不存在"
**解决**：插入管理员数据
```sql
INSERT INTO admin (username, password, real_name, role, status, create_time, is_deleted)
VALUES ('admin', '$2a$10$N.zmdr9k7uOCQv37Y7Cb./8iB6W9W9W9W9W9W9W9W9W9W9W9W9W9W9', '超级管理员', 1, 1, NOW(), 0);
```

## 密码加密说明

BCrypt加密的密码格式：
- 明文：`admin123`
- 密文：`$2a$10$N.zmdr9k7uOCQv37Y7Cb./8iB6W9W9W9W9W9W9W9W9W9W9W9W9W9W9`

**注意**：每次加密同样的密码会得到不同的密文，但验证时都能通过。

4. 查看响应数据是否正确
