### 9.2 降级方案详情

#### 方案A: 知识图谱Canvas降级（触发条件: R1发生）

```
正常方案: Canvas2D绘制49节点 + 99边 + 手势交互(缩放/拖拽/单击/双击)
         ↓ 性能不达标时
降级方案: 
  Level 1 (轻度): depth默认值从2改为1 → 只显示直接相邻节点(~15个)
           → FPS从18提升到30+
           
  Level 2 (中度): 移除连线动画效果 → 静态线条
           → 渲染时间减少40%
           
  Level 3 (重度): 切换到列表模式(替代Canvas)
           → 节点以卡片列表展示(类似药材列表)
           → 点击节点→展开关联节点子列表(手风琴折叠)
           → 完全放弃可视化，保留数据完整性
           
  Level 4 (终极): 使用预渲染图片(静态PNG/SVG)
           → 后端预生成几张常见中心节点的图谱图片
           → 前端直接<img src="graph_1.png">显示
           → 无交互，只读展示

决策权: A在Day13 15:30评估后决定采用哪个Level
```

#### 方案B: AI接口降级（触发条件: R2发生）

```
正常方案: 实时调用DeepSeek API → 动态生成回复
         ↓ 连续超时3次或返回429(限流)
降级方案:
  Step 1: 超时阈值从10s降到5s → 快速失败
  Step 2: 返回预置Q&A缓存匹配(关键词匹配):
          - "高血压" → 缓存回答A(关于高血压的建议)
          - "糖尿病" → 缓存回答B
          - "康复训练" → 缓存回答C
          ...共50条覆盖常见场景
          
  Step 3: UI提示优化:
          - 正常: 显示"AI思考中..." + Loading动画
          - 降级: 显示"智能推荐"(不提AI) + 缓存内容
          - 用户无感知差异!(除非问非常偏门的问题)

决策权: C实时判断(无需开会)，B配合UI文案修改
```
