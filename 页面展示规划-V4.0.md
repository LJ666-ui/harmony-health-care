### 1.2 轮播Banner设计规范

**统一规格**：
- 尺寸：宽度100% × 高度180px（首页）/ 160px（子Tab）
- 自动轮播：间隔3秒，支持手动滑动
- 圆角：12px，阴影：0 4px 12px rgba(0,0,0,0.1)
- 内容结构：左侧图标+标题+描述 → 右侧渐变背景色
- 点击事件：跳转对应功能详情页

**颜色主题轮换**：
```
Banner 1: 渐变蓝 (#4A90E2 → #357ABD)   → 医疗专业感
Banner 2: 渐变绿 (#50C878 → #3DA35D)   → 健康活力感
Banner 3: 渐变紫 (#9B59B6 → #8E44AD)   → AI科技感
Banner 4: 渐变橙 (#F39C12 → #E67E22)   → 温馨服务感
Banner 5: 渐变青 (#1ABC9C → #16A085)   → 智能创新感
```

## 二、现状诊断与优化方案

### 2.1 首页功能卡片过载问题（已解决 🔴✅）

**原问题**：HomePage包含14个功能入口，信息密度过高

**V4.0解决方案**：精简为 **6个高频入口 + 3个推荐位**

#### 精简后的首页快捷入口（2×3网格）

| 位置 | 图标 | 入口名称 | 跳转目标 | 选择理由 |
|-----|------|---------|---------|---------|
| 左上 | 📋 | 健康记录 | HealthRecords | 核心功能，每日必查 |
| 中上 | 🤖 | AI健康顾问 | AiChatPage (V3.0功能7增强) | 差异化亮点，高频使用 |
| 右上 | 📅 | 我的预约 | Appointments | 就医流程关键节点 |
| 左下 | 💊 | 用药提醒 | Medications | 老年用户刚需 |
| 中下 | 🎤 | 语音助手 | VoiceAssistantPage | 无障碍核心能力 |
| 右下 | 🅿️ | 智慧停车 | ParkingLotPage (V3.0功能2) | 医院场景特色功能 |

#### 移除的8个入口及其去向

| 移除入口 | 新归属位置 | 说明 |
|---------|----------|------|
| 饮食食谱 | 「发现」Tab → 食材百科板块 | 归入内容聚合区 |
| 药材百科 | 「发现」 Tab → 中药材板块 | 归入内容聚合区 |
| 设备管理 | 「我的」Tab → 设备管理入口 | 个人设置区 |
| 隐私设置 | 「我的」Tab → 设置列表 | 个人设置区 |
| 转院审批 | 「就医」Tab → 更多服务折叠区 | 低频功能收起 |
| 实时监控 | 「健康」Tab → 实时监控Tab | 专业功能区 |
| 影像分析 | 「发现」 Tab → AI工具箱 | AI工具集中区 |
| AI风险评估 | 「健康」 Tab → 风险评估Tab | 专业功能区 |

### 2.2 主题系统混用问题（已解决 🔴✅）

**原问题**：GlobalTheme（旧）vs AccessibilityConfig（新）两套体系并存

**V4.0解决方案**：全面迁移至 `AccessibilityConfig`

**迁移优先级**：
- P0: MedicalPage.ets, RehabPage.ets, SciencePage.ets （~170处调用）
- P1: HospitalPage.ets, DepartmentPage.ets 及其余所有子页面

**API对照表**（完整版见第七章）

## 六、主题统一迁移方案（V4.0 完整版）

### 6.1 迁移目标

**统一使用**: `AccessibilityConfig`（位于 `utils/AccessibilityConfig.ets`）
**废弃**: `GlobalTheme`（位于 `global.ets`），逐步替换后删除

### 6.2 API对照表（完整版）

| 功能 | GlobalTheme（旧） | AccessibilityConfig（新） | 迁移示例 |
|-----|-------------------|--------------------------|---------|
| 主题色 | `GlobalTheme.THEME_COLOR` | `AccessibilityConfig.getThemeColor()` | `.backgroundColor(GlobalTheme.THEME_COLOR)` → `.backgroundColor(AccessibilityConfig.getThemeColor())` |
| 标题字号 | `GlobalTheme.getFontSizeTitle(isOld)` | `AccessibilityConfig.getFontSizeTitle()` | `.fontSize(GlobalTheme.getFontSizeTitle(this.isOld))` → `.fontSize(AccessibilityConfig.getFontSizeTitle())` |
| 正文字号 | `GlobalTheme.getFontSizeBody(isOld)` | `AccessibilityConfig.getFontSizeBody()` | 同上模式 |
| 小字字号 | `GlobalTheme.getFontSizeSmall(isOld)` | `AccessibilityConfig.getFontSizeSmall()` | 同上模式 |
| 大间距 | `GlobalTheme.getSpacingLG(isOld)` | `AccessibilityConfig.getSpacingLG()` | `.margin({ top: GlobalTheme.getSpacingLG(this.isOld) })` → `.margin({ top: AccessibilityConfig.getSpacingLG() })` |
| 中间距 | `GlobalTheme.getSpacingMD(isOld)` | `AccessibilityConfig.getSpacingMD()` | 同上模式 |
| 小间距 | `GlobalTheme.getSpacingSM(isOld)` | `AccessibilityConfig.getSpacingSM()` | 同上模式 |
| 圆角 | `GlobalTheme.RADIUS_MD` | `AccessibilityConfig.getRadius()` | `.borderRadius(GlobalTheme.RADIUS_MD)` → `.borderRadius(AccessibilityConfig.getRadius())` |
| 卡片背景色 | `'#FFFFFF'` (硬编码) | `AccessibilityConfig.getCardBgColor()` | `.backgroundColor('#FFFFFF')` → `.backgroundColor(AccessibilityConfig.getCardBgColor())` |
| 边框色 | `GlobalTheme.BORDER_COLOR` | `AccessibilityConfig.getBorderColor()` | `.borderColor(...)` 替换 |
| 主文本色 | `'#333333'` (硬编码) | `AccessibilityConfig.getTextPrimary()` | `.fontColor('#333333')` → `.fontColor(AccessibilityConfig.getTextPrimary())` |
| 次文本色 | `GlobalTheme.TEXT_SECONDARY` | `AccessibilityConfig.getTextSecondary()` | 替换 |
| 提示文本色 | `GlobalTheme.TEXT_HINT` | `AccessibilityConfig.getTextHint()` | 替换 |
| 背景色 | `GlobalTheme.BG_COLOR` | `AccessibilityConfig.getBgColor()` | 替换 |

## 十二、总结

### V4.0 核心价值主张

✅ **解决痛点**：首页信息过载（14→6）、主题混用、导航混乱、V3.0功能零集成

✅ **提升体验**：22个动态轮播Banner带来视觉活力和运营灵活性，清晰的信息架构降低认知负荷

✅ **技术债务清理**：统一主题系统、组件化重构、消除重复代码，为后续迭代打下坚实基础

✅ **商业化赋能**：22个Banner位 + 会员权益展示 = 可观的运营推广和变现空间

✅ **创新功能落地**：V3.0的10个前沿功能（分布式协同/AI风险/AR导航/数字孪生等）终于有了合适的展示舞台

4. **集中攻坚** Phase 3（四个Tab页强化），这是工作量最大的阶段
5. **充分测试** Phase 5（联调测试），确保质量后再合并主分支
