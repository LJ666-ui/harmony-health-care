# 星云医疗助手 - 页面展示规划方案 v2.0

> 项目路径：`harmony-health-care`
> 技术栈：HarmonyOS ArkTS (ETS) + Spring Boot
> 规划日期：2026-04-21
> 页面总数：73个 | 组件数：39个 | 后端Controller：26个

---

## 一、现状诊断：当前页面存在的核心问题

### 1.1 首页功能卡片过载（严重 🔴）

**问题**：[HomePage.ets](entry/src/main/ets/pages/HomePage.ets) 当前包含 **14个** 功能入口卡片，信息密度过高。

| 现有14个入口 | 归属模块 | 问题 |
|-------------|---------|------|
| 📋 健康记录 | 首页核心 | ✅ 保留 |
| 📅 预约管理 | 就医服务 | ⚠️ 与就医Tab重复 |
| 💊 用药管理 | 就医服务 | ⚠️ 与就医Tab重复 |
| 🍎 饮食食谱 | 科普内容 | ⚠️ 与科普Tab重复 |
| 🤖 AI健康顾问 | AI服务 | ✅ 保留（高频） |
| 🎤 语音助手 | 个人中心 | ⚠️ 与我的Tab重复 |
| 🌿 药材百科 | 科普内容 | ⚠️ 与科普Tab重复 |
| 📱 设备管理 | 个人中心 | ⚠️ 与我的Tab重复 |
| 🔒 隐私设置 | 个人中心 | ❌ 不应在首页 |
| 🔄 转院审批 | 就医服务 | ❌ 低频功能 |
| 🅿️ 智慧停车 | 停车服务 | ❌ 独立业务线 |
| 🧠 AI风险评估 | 康复服务 | ⚠️ 与康复Tab重复 |
| 📊 实时监控 | 监控系统 | ❌ 专业功能 |
| 🔬 影像分析 | AI服务 | ❌ 低频专业 |

**建议**：首页精简为 **6-8个** 高频入口，其余归入各Tab子页面。

---

### 1.2 主题系统混用（严重 🔴）

**问题**：项目同时存在两套主题配置体系，不同页面使用不同的主题API：

| 页面 | 使用主题 | 文件位置 |
|-----|---------|---------|
| [MedicalPage.ets](entry/src/main/ets/pages/MedicalPage.ets) | `GlobalTheme`（旧版） | `global.ets` |
| [RehabPage.ets](entry/src/main/ets/pages/RehabPage.ets) | `GlobalTheme`（旧版） | `global.ets` |
| [SciencePage.ets](entry/src/main/ets/pages/SciencePage.ets) | `GlobalTheme`（旧版） | `global.ets` |
| [Profile.ets](entry/src/main/ets/pages/Profile.ets) | `AccessibilityConfig`（新版） | `utils/AccessibilityConfig.ets` |
| [Footer.ets](entry/src/main/ets/components/Footer.ets) | `AccessibilityConfig`（新版） | `utils/AccessibilityConfig.ets` |
| [HomePage.ets](entry/src/main/ets/pages/HomePage.ets) | `AccessibilityConfig`（新版） | `utils/AccessibilityConfig.ets` |

**影响**：
- 字号、间距、颜色在不同页面间不一致
- 适老化模式切换效果不统一
- 维护成本高，新增页面不知用哪套

**建议**：统一迁移至 `AccessibilityConfig` 新版主题体系。

## 二、规划方案：全新页面架构

### 2.1 设计原则

1. **首页做减法** — 只保留最高频的6个快捷入口 + 健康数据概览
2. **各Tab做加法** — 让每个Tab成为该领域的完整服务中心
3. **统一主题** — 全部迁移到 AccessibilityConfig
4. **补全路由** — 所有可访问页面必须注册
5. **适老化优先** — 所有页面默认支持老年模式

### 2.2 底部导航重构（保持5 Tab）

```
┌─────────────────────────────────────────────────────────┐
│  底部Tab导航栏                                           │
├──────┬──────┬──────┬──────┬─────────────────────────────┤
│ 首页 │ 就医 │ 健康 │ 发现 │ 我的                        │
│ 🏠   │ 🏥   │ ❤️   │ 🔍   │ 👤                          │
└──────┴──────┴──────┴──────┴─────────────────────────────┘
```

> **变更说明**："康复"→"健康"，"科普"→"发现"，语义更清晰，覆盖面更广

### 2.3 各页面详细规划

## 五、主题统一方案

### 5.1 迁移目标

**统一使用**：`AccessibilityConfig`（位于 `utils/AccessibilityConfig.ets`）

**废弃**：`GlobalTheme`（位于 `global.ets`），逐步替换后删除

### 5.2 API对照表

| 功能 | GlobalTheme（旧） | AccessibilityConfig（新） |
|-----|-------------------|--------------------------|
| 主题色 | `GlobalTheme.THEME_COLOR` | `AccessibilityConfig.getThemeColor()` |
| 标题字号 | `GlobalTheme.getFontSizeTitle(isOld)` | `AccessibilityConfig.getFontSizeTitle()` |
| 正文字号 | `GlobalTheme.getFontSizeBody(isOld)` | `AccessibilityConfig.getFontSizeBody()` |
| 大间距 | `GlobalTheme.getSpacingLG(isOld)` | `AccessibilityConfig.getSpacingLG()` |
| 中间距 | `GlobalTheme.getSpacingMD(isOld)` | `AccessibilityConfig.getSpacingMD()` |
| 圆角 | `GlobalTheme.RADIUS_MD` | `AccessibilityConfig.getRadius()` |
| 卡片背景色 | `'#FFFFFF'` (硬编码) | `AccessibilityConfig.getCardBgColor()` |
| 边框色 | `GlobalTheme.BORDER_COLOR` | `AccessibilityConfig.getBorderColor()` |
| 主文本色 | `'#333333'` (硬编码) | `AccessibilityConfig.getTextPrimary()` |
| 次文本色 | `GlobalTheme.TEXT_SECONDARY` | `AccessibilityConfig.getTextSecondary()` |
