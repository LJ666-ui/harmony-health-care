```
┌─────────────────────────────────────────────────────────────┐

                            ▼

└───────────────────────────┬─────────────────────────────────┘
                            │ HTTPS
                            ▼
┌─────────────────────────────────────────────────────────────┐
│                      🤖 AI服务层 (硅基流动)                    │
│  DeepSeek-V3 API: 咨询/评估解读/康复指导/图谱探索/图像复原     │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 关键技术选型理由

| 技术选型 | 选择原因 | 替代方案(未采用) |
|---------|----------|----------------|
| **HarmonyOS NEXT** | 华为生态/分布式能力/官方支持 | Flutter/React Native |
| **ArkTS** | 鸿蒙原生语言/性能优秀 | JavaScript/TypeScript |
| **Spring Boot** | Java生态成熟/团队熟悉 | Node.js/Go |
| **MyBatis-Plus** | 简化CRUD/代码生成 | JPA/Hibernate |
| **MySQL** | 关系型/事务支持/免费 | PostgreSQL/MongoDB |
| **SQLite(前端)** | 鸿蒙内置/离线优先/轻量 | Realm/IndexedDB |
| **DeepSeek API** | 中文能力强/成本低/响应快 | GPT-4/Claude |

### 3.3 SQLite 表清单（8张）— 全部新建！

| # | 表名 | 用途 | 数据来源 | 清理策略 |
|---|------|------|---------|---------|
| 1 | **local_user_cache** | 登录状态缓存(Token/用户信息) | 登录成功后写入 | 永远只有1条 |
| 2 | **offline_health_record** | 离线健康数据队列 | 断网时手动输入 | 同步后清理 |
| 3 | **local_article_cache** | 文章阅读缓存(标题/摘要) | 打开文章时写入 | 7天过期/上限200条 |
| 4 | **local_device_cache** | 鸿蒙设备配对缓存 | 配对成功后写入 | 手动解绑删除 |
| 5 | **sync_queue** | 数据同步任务队列 | 离线操作自动入队 | 同步成功后删除 |
| 6 | **local_browse_history** ⭐ | 本地浏览历史记录 | BrowseTracker自动追踪 | 只保留30天 |
| 7 | **local_search_history** | 本地搜索历史(下拉提示) | 搜索框提交时记录 | 去重后保留100条 |
| 8 | **local_collection_cache** | 本地收藏夹缓存 | 点击收藏时双写 | 取消收藏时删除 |

### 5.2 V7.1 四大新创新点详解

#### 创新点7: 药材百科展示

```
技术实现:
  前端: WaterfallGrid(瀑布流) + CategoryTabs(分类) + HerbalCard(卡片)
        + EfficacyTag(功效标签云) + SearchBar(全文搜索)
  后端: herbal_medicine(200条) + herbal_efficacy(~500条) + MyBatis-Plus分页
  特色: 中文/拼音/别名三维搜索 + 四属性筛选 + 功效可视化 + 方剂关联 + 收藏双写
```

#### 创新点8: 医学知识图谱

```
技术实现:
  前端Canvas: GraphCanvas(drawNode/drawEdge/forceLayout/handleGesture)
              + KnowledgeNode + NodeDetailPanel
  后端算法: knowledge_node(49节点: 8疾病+12症状+20药材+6方剂+3疗法)
            + knowledge_edge(99条边: has_symptom/treats/alleviates/contains...)
            + BFS子图展开算法
  AI增强: 自然语言→子图探索 ("高血压吃什么中药?" → 解析→BFS→可视化+解读)
```

#### 创新点9 & 10: 推荐引擎 + 离线架构

```
数据流:
  用户浏览 → BrowseTracker(全局埋点) → local_browse_history(SQLite,纯本地!)
          → RecommendEngine(统计分析) → Top N高频对象 → DailyRecommendPage

离线模式:
  有网: App操作 → 即时显示 → 异步POST服务器
  断网: App操作 → offline_health_record + sync_queue → Toast"已保存"
  恢复: SyncManager消费队列 → POST服务器 → 更新synced=1 → "N条已同步"
```
