## 一、项目概述

### 1.1 项目定位
**鸿蒙医疗健康管理系统 (Harmony Medical Operating System, HMOS)** 是一个面向多角色的智慧医疗健康管理平台，支持鸿蒙设备生态接入，集成AI智能服务与中医知识图谱。

### 1.2 技术架构

| 层级 | 技术选型 |
|------|----------|
| **后端框架** | Spring Boot 2.7.16 + Java 8 |
| **ORM框架** | MyBatis-Plus 3.5.3 |
| **数据库** | MySQL 8.0 |
| **前端框架** | Vue 3 + Vite 5 |
| **UI组件库** | Element Plus 2.4 |
| **状态管理** | Pinia 2.1 |
| **图表可视化** | ECharts 5.4 |
| **认证机制** | JWT + BCrypt密码加密 |
| **AI集成** | FastJSON + AI客户端（大模型对接）|
| **API文档** | Swagger/SpringDoc OpenAPI |
| **国际化** | vue-i18n (中/英双语) |

## 二、高优先级扩展建议

### 2.1 🔥 在线问诊/视频诊疗系统

#### 业务价值
当前系统已有预约和AI咨询功能，但缺少**实时医患沟通**能力。视频问诊是互联网医疗的核心场景。

#### 扩展方案

**后端新增：**
```java
// 新增模块结构
controller/
  ├── ConsultationController.java      // 问诊控制器
  └── VideoCallController.java         // 视频通话信令

service/
  ├── ConsultationService.java         // 问诊服务
  └── WebSocketService.java            // 实时通信服务

entity/
  ├── Consultation.java                // 问诊记录实体
  ├── ChatMessage.java                 // 聊天消息实体
  └── VideoSession.java                // 视频会话实体
```

**数据库表设计：**
```sql
-- 问诊订单表
CREATE TABLE consultation_order (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    order_no VARCHAR(64) UNIQUE COMMENT '问诊单号',
    patient_id BIGINT NOT NULL COMMENT '患者ID',
    doctor_id BIGINT NOT NULL COMMENT '医生ID',
    department VARCHAR(100) COMMENT '就诊科室',
    consultation_type TINYINT DEFAULT 1 COMMENT '类型:1=图文 2=语音 3=视频',
    status TINYINT DEFAULT 0 COMMENT '状态:0待接诊 1进行中 2已完成 3已取消',
    chief_complaint TEXT COMMENT '主诉描述',
    diagnosis_result TEXT COMMENT '诊断结果',
    prescription_id BIGINT COMMENT '关联处方ID',
    fee DECIMAL(10,2) COMMENT '问诊费用',
    start_time DATETIME COMMENT '开始时间',
    end_time DATETIME COMMENT '结束时间',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 聊天消息表
CREATE TABLE chat_message (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    consultation_id BIGINT NOT NULL COMMENT '问诊单ID',
    sender_id BIGINT NOT NULL COMMENT '发送者ID',
    sender_type TINYINT COMMENT '发送者类型:1=患者 2=医生',
    message_type TINYINT DEFAULT 1 COMMENT '消息类型:1=文本 2=图片 3=语音 4=处方',
    content TEXT COMMENT '消息内容',
    media_url VARCHAR(500) COMMENT '媒体文件URL',
    is_read TINYINT DEFAULT 0 COMMENT '是否已读',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

**技术实现要点：**
- 使用 **WebSocket (STOMP)** 实现实时消息推送
- 集成 **WebRTC** 实现浏览器端视频通话
- 支持 **图文/语音/视频** 三种问诊模式
- 问诊结束后自动生成电子病历摘要
- 支持在线开具电子处方并流转到药房

#### 前端页面扩展
- `/user/consultation` - 患者端问诊列表与聊天窗口
- `/doctor/consultation` - 医生端候诊队列与接诊界面
- 集成 WebRTC 视频通话组件
- 支持图片/语音/文件上传

### 2.3 💊 智能药房与药品管理系统增强

#### 业务价值
现有 `medicine` 表较简单，缺乏完整的**药品流通链路**管理。

#### 扩展方案

**新增业务流程：**
```
药品入库 → 库存管理 → 处方开立 → 用药审核 → 发药确认 → 用药提醒 → 用药反馈
```

**数据库扩展：**
```sql
-- 药品库存表
CREATE TABLE medicine_inventory (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    medicine_id BIGINT NOT NULL COMMENT '药品ID',
    hospital_id BIGINT COMMENT '医院ID',
    batch_no VARCHAR(64) COMMENT '批次号',
    stock_quantity INT DEFAULT 0 COMMENT '当前库存',
    warning_quantity INT DEFAULT 10 COMMENT '预警数量',
    expiry_date DATE COMMENT '有效期',
    supplier VARCHAR(200) COMMENT '供应商',
    price DECIMAL(10,2) COMMENT '采购价',
    selling_price DECIMAL(10,2) COMMENT '售价',
    status TINYINT DEFAULT 1 COMMENT '状态:1正常 2停用'
);

-- 电子处方表(增强)
CREATE TABLE electronic_prescription (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    prescription_no VARCHAR(64) UNIQUE COMMENT '处方编号',
    patient_id BIGINT NOT NULL COMMENT '患者ID',
    doctor_id BIGINT NOT NULL COMMENT '开方医生ID',
    diagnosis TEXT COMMENT '诊断',
    items JSON COMMENT '处方明细[{medicineId, quantity, usage, frequency}]',
    total_amount DECIMAL(10,2) COMMENT '总金额',
    status TINYINT DEFAULT 0 COMMENT '状态:0待审核 1已审核 2已发药 3已取药',
    auditor_id BIGINT COMMENT '审核药师ID',
    pharmacist_id BIGINT COMMENT '发药药师ID',
    dispense_time DATETIME COMMENT '发药时间',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 用药计划表
CREATE TABLE medication_schedule (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    prescription_id BIGINT COMMENT '关联处方',
    patient_id BIGINT NOT NULL COMMENT '患者ID',
    medicine_name VARCHAR(100) COMMENT '药品名称',
    dosage VARCHAR(50) COMMENT '剂量',
    frequency VARCHAR(50) COMMENT '频率',
    start_date DATE COMMENT '开始日期',
    end_date DATE COMMENT '结束日期',
    reminder_times JSON COMMENT '提醒时间点["08:00","12:00","18:00"]',
    is_active TINYINT DEFAULT 1 COMMENT '是否生效'
);
```

**功能增强点：**
- 药品效期自动预警与临期提醒
- 处方审核流程（医生开方→药师审核→发药）
- 智能用药提醒（结合设备推送）
- 药物相互作用检测（基于AI知识图谱）
- 用药依从性统计分析

```sql
-- 结构化病历主表
```
