## 🐛 常见问题排查

### 问题1：后端启动失败 - 数据库连接错误
**错误信息**:
```
Caused by: com.mysql.cj.jdbc.exceptions.CommunicationsException: 
Communications link failure
```

**解决方案**:
```bash
# 1. 确保MySQL服务正在运行
# Windows: Win+R → services.msc → 找到MySQL → 启动

# 2. 检查数据库配置
# 文件: src/main/resources/application.yml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/medical_health?...
    username: root
    password: 123456  # ← 确认密码正确

# 3. 测试数据库连接
mysql -u root -p123456 -e "SELECT COUNT(*) FROM medical_health.health_food;"
# 预期输出: 743
```

### 问题2：手机无法访问电脑的8080端口
**错误信息**:


**解决方案**:

```bash
# 1. 打开控制面板 → 系统和安全 → Windows Defender 防火墙

# 3. 选择"关闭Windows Defender防火墙（不推荐）"

```

#### B. 添加防火墙入站规则（推荐）
```bash
# 以管理员身份运行PowerShell
```

- 暂时禁用360安全卫士、腾讯电脑管家等
- 或添加Java进程到白名单

## 💡 一键启动脚本（Windows）

创建文件 `start-backend.bat` 放在项目根目录：

```batch
@echo off
chcp 65001 >nul
echo ========================================
echo   医疗健康系统 - 后端服务启动工具
echo ========================================
echo.

echo [1/4] 检查Java环境...
java -version >nul 2>&1
if errorlevel 1 (
    echo ❌ 错误: 未检测到Java，请先安装JDK 8+
    pause
    exit /b 1
)
echo ✅ Java环境正常

echo.
echo [2/4] 检查MySQL连接...
mysql -u root -p123456 -e "SELECT 1" >nul 2>&1
if errorlevel 1 (
    echo ⚠️  警告: 无法连接MySQL，请确认MySQL服务已启动
    echo    数据库: localhost:3306/medical_health
    echo    用户名: root, 密码: 123456
) else (
    echo ✅ MySQL连接正常
)

echo.
echo [3/4] 检查8080端口...
netstat -ano | findstr ":8080.*LISTENING" >nul 2>&1
if not errorlevel 1 (
    echo ⚠️  警告: 8080端口已被占用，可能是旧的后端进程
    echo    尝试自动关闭...
    for /f "tokens=5" %%a in ('netstat -ano ^| findstr ":8080.*LISTENING"') do taskkill /F /PID %%a >nul 2>&1
    timeout /t 2 >nul
) else (
    echo ✅ 8080端口空闲
)

echo.
echo [4/4] 启动后端服务...
echo ========================================
echo   正在启动Spring Boot应用...
echo   请等待看到 "Started MedicalApplication" 提示
echo   按 Ctrl+C 可停止服务
echo ========================================

cd /d %~dp0
call mvn spring-boot:run

pause
```

**使用方法**:
1. 双击 `start-backend.bat`
2. 等待看到 `Started MedicalApplication`
3. 不要关闭窗口，保持后端运行
4. 在手机上测试食材功能
