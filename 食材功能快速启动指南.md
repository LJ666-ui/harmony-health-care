## ❓ 常见问题排查

### 问题1：start-backend.bat 启动失败

**错误信息**: `'mvn' 不是内部或外部命令`

**解决方案**:
```bash
# 方案1: 安装Maven
1. 下载: https://maven.apache.org/download.cgi
2. 解压到: C:\Program Files\Apache\maven
3. 添加环境变量:
   - 变量名: MAVEN_HOME
   - 变量值: C:\Program Files\Apache\maven
   - Path添加: %MAVEN_HOME%\bin
4. 重启CMD，再次运行脚本

# 方案2: 使用IDEA启动
直接用IDEA打开项目，运行MedicalApplication.java
```

---

### 问题2：MySQL连接失败

**错误信息**: `Could not create connection to database server`

**解决方案**:
```bash
# 1. 启动MySQL服务
Win+R → services.msc → 找到MySQL → 右键 → 启动

# 2. 或者通过任务管理器
Ctrl+Shift+Esc → 服务选项卡 → MySQL → 启动

# 3. 验证连接
mysql -u root -p123456 -e "SELECT VERSION();"
# 应该显示版本号，如: 8.0.33
```

---

### 问题3：8080端口被占用

**错误信息**: `Port 8080 was already in use`

**解决方案**:
```bash
# 方法1: 结束占用进程（脚本会自动提示）
netstat -ano | findstr ":8080.*LISTENING"
taskkill /F /PID <PID号>

# 方法2: 使用其他端口
mvn spring-boot:run -Dserver.port=8081
# 同时需要修改前端ApiConstants.ets中的端口号为8081
```

---

### 问题4：前端编译错误

**现象**: DevEco Studio显示红色错误

**解决方案**:
```
1. 查看具体的错误信息（鼠标悬停在红线上）
2. 常见错误类型:
   - 缺少依赖: File → Project Structure → Dependencies → +
   - 类型错误: 查看之前的ArkTS编译错误修复
   - 导入缺失: 添加正确的import语句

3. 如果是之前修复的类型错误未保存:
   - 确保所有修改的.ets文件都已保存 (Ctrl+S)
   - 重新Build → Rebuild Project
```

---

### 问题5：模拟器中仍显示"暂无数据"

**可能原因及检查顺序**:

```
1️⃣ 后端真的启动了吗？
   → 运行 test-backend.bat 检测
   → 或浏览器访问 http://localhost:8080/test-health-food

2️⃣ 模拟器的网络设置对吗？
   → Android模拟器的localhost = 10.0.2.2
   → 已在ApiConstants.ets中配置正确 ✅

3️⃣ 防火墙阻止了吗？
   → 临时关闭Windows防火墙测试
   → 或添加Java入站规则

4️⃣ 前端代码是最新的吗？
   → 确保已Rebuild Project
   → 重新安装应用到模拟器

5️⃣ 查看Logcat日志
   → DevEco Studio底部的Logcat面板
   → 过滤关键字: HttpUtil 或 FoodEncyclopedia
   → 查看是否有网络请求和错误信息
```

```
┌──────────────┐                              ┌──────────────────┐

└──────────────┘                              └────────┬─────────┘
                                                      │
```

**请求URL**: `http://10.0.2.2:8080/healthFood/page?page=1&size=20`

**关键点**:
- `10.0.2.2` = Android模拟器访问宿主机（您的电脑）的特殊IP
- `8080` = Spring Boot监听的端口
- `/healthFood/page` = 食材分页接口路径
- `?page=1&size=20` = 第1页，每页20条
