## 📌 一、项目是什么？

**星云医疗助手**是给老年人用的医疗健康APP后端系统：
- 老人记录血压、血糖等健康数据 → 系统自动评估风险
- 提供康复训练指导、医院查询、科普文章学习
- 医生查看病历、跨院转诊
- 数据通过鸿蒙分布式能力在多设备间安全同步

**你的任务**：搭建整个后端服务，让前端APP能正常调用这些功能。

### 🔍 每个目录是干什么的？

#### **1️⃣ controller/** - 你的"前台接待员"

**职责**：
- 定义API路径（如 `POST /user/login`）
- 接收前端传来的参数（从HTTP请求中提取）
- 参数校验（检查必填项、格式是否正确）
- 调用Service处理业务
- 返回统一格式的JSON响应

**示例代码**：
```java
@RestController
@RequestMapping("/user")
public class UserController {
    
    @Autowired
    private UserService userService;
    
    @PostMapping("/login")
    public Result<?> login(@RequestBody User user) {
        // 1. 校验参数
        if (user.getUsername() == null || user.getPassword() == null) {
            return Result.error("用户名和密码不能为空");
        }
        
        // 2. 调用Service层
        User result = userService.login(user);
        
        // 3. 返回结果
        return Result.success("登录成功", result);
    }
}
```

#### **3️⃣ mapper/** - 你的"数据库管家"

**职责**：
- 执行SQL语句与MySQL交互

userMapper.selectList(null);          // 查询全部
userMapper.selectPage(page, wrapper); // 分页查询

    private String password;      // BCrypt加密后的密码
    
    @TableLogic
    private Integer isDeleted;    // 逻辑删除标志
    
    private Date createTime;      // 创建时间
    private Date updateTime;      // 更新时间
}

#### **5️⃣ common/** - 公共工具（🔴🔴🔴 极其重要！）

##### **Result.java** - 统一响应格式

**这是你写的每个接口都必须返回的格式！**

```java
public class Result<T> {
    private int code;        // 200=成功 400=参数错误 401=未授权 500=服务器错误
    private String message;  // 提示信息
    private T data;          // 实际数据
}

// 使用方式
return Result.success("登录成功", userMap);     // 成功
return Result.error("用户名已存在");             // 失败
return Result.success("删除成功");              // 成功无数据
```

**为什么重要？**
- 前端只需判断 `code == 200` 就知道是否成功
- 统一错误码体系，方便问题排查
- 包含泛型 `<T>`，可返回任意类型

##### **⭐ RiskAssessmentServiceImpl.java** - 风险评估算法核心

**这是什么？**

**有什么用？**

public class RiskAssessmentServiceImpl implements RiskAssessmentService {

    @Autowired
    private HealthRecordService healthRecordService;
    
    @Autowired
    private AIService aiService;  // 调用AI生成建议
    
    /**
     * 主入口：执行完整的5种疾病风险评估
     */
    public RiskAssessmentResult assessAllRisks(Long userId) {
        // 1. 获取用户最近30天的健康数据
        List<HealthRecord> records = healthRecordService.getLast30Days(userId);
        
        if (records.isEmpty()) {
            throw new BusinessException("暂无足够的健康数据进行评估");
        }
        
        // 2. 分别计算5种疾病的风险
        Map<String, RiskItem> risks = new HashMap<>();
        risks.put("hypertension", calculateHypertensionRisk(records));    // 高血压
        risks.put("diabetes", calculateDiabetesRisk(records));          // 糖尿病
        risks.put("cvd", calculateCVDRisk(records));                    // 心血管疾病
        risks.put("fall_risk", calculateFallRisk(records));            // 跌倒风险
        risks.put("cognitive", calculateCognitiveRisk(records));       // 认知障碍
        
        // 3. 计算综合风险等级
        String overallLevel = calculateOverallLevel(risks);
        
        // 4. 调用AI生成个性化建议
        String aiAdvice = aiService.generateHealthAdvice(risks);
        
        // 5. 构建返回结果并保存到数据库
        RiskAssessmentResult result = new RiskAssessmentResult(
            userId, risks, overallLevel, aiAdvice
        );
        
        riskAssessmentMapper.insert(result);  // 保存历史记录
        
        return result;
    }
    
    /**
     * 高血压风险评估算法
     * 规则：
     * - 收缩压 ≥ 180 或 舒张压 ≥ 110 → critical (90-100分)
     * - 收缩压 140-179 或 舒张压 90-109 → high (70-89分)
     * - 收缩压 120-139 或 舒张压 80-89 → medium (40-69分)
     * - 收缩压 < 120 且 舒张压 < 80 → low (0-39分)
     */
    private RiskItem calculateHypertensionRisk(List<HealthRecord> records) {
        List<Integer> systolicList = new ArrayList<>();  // 收缩压
        List<Integer> diastolicList = new ArrayList<>(); // 舒张压
        
        for (HealthRecord record : records) {
            if ("blood_pressure".equals(record.getType())) {
                String[] values = record.getValue().split("/");
                systolicList.add(Integer.parseInt(values[0]));
                diastolicList.add(Integer.parseInt(values[1]));
            }
        }
        
        int avgSystolic = average(systolicList);
        int avgDiastolic = average(diastolicList);
        
        int score;
        String level;
        
        if (avgSystolic >= 180 || avgDiastolic >= 110) {
            score = 95; level = "critical";
        } else if (avgSystolic >= 140 || avgDiastolic >= 90) {
            score = 80; level = "high";
        } else if (avgSystolic >= 120 || avgDiastolic >= 80) {
            score = 55; level = "medium";
        } else {
            score = 20; level = "low";
        }
        
        return new RiskItem("hypertension", "高血压", score, level,
            String.format("近期平均血压：%d/%d mmHg", avgSystolic, avgDiastolic));
    }

     * 糖尿病风险评估算法

     * - ≥ 7.0 mmol/L → high (70-100分)

##### **⭐ AIServiceImpl.java** - DeepSeek AI集成

**这是什么？**
连接DeepSeek大模型API的服务层，为前端提供智能问答能力。

**有什么用？**
- 回答老年人的健康咨询问题
- 基于风险评估结果生成个性化建议
- 解读复杂的健康数据趋势

**实现了什么功能？**

```java
@Service
public class AIServiceImpl implements AIService {

    @Value("${deepseek.api-key}")
    private String apiKey;
    
    @Value("${deepseek.api-url}")
    private String apiUrl;
    
    /**
     * 与AI对话
     */
    public AIChatResponse chatWithAI(AIChatRequest request) {
        // 1. 构建系统提示词（医疗场景专用）
        String systemPrompt = buildSystemPrompt(request.getUserId());
        
        // 2. 构建用户消息
        List<Message> messages = Arrays.asList(
            new Message("system", systemPrompt),
            new Message("user", request.getQuestion())
        );
        
        // 3. 调用DeepSeek API
        String response = callDeepSeekAPI(messages);
        
        // 4. 解析响应并返回
        return new AIChatResponse(response);
    }
    
    /**
     * 生成健康建议（基于风险评估结果）
     */
    public String generateHealthAdvice(Map<String, RiskItem> risks) {
        StringBuilder prompt = new StringBuilder();
        prompt.append("您是一位专业的老年健康管理专家。");
        prompt.append("\n根据以下风险评估结果，请用通俗易懂的语言给出健康建议：\n\n");
        
        for (RiskItem risk : risks.values()) {
            prompt.append(String.format("- %s（%s）：%s\n", 
                risk.getName(), risk.getLevel(), risk.getDescription()));
        }
        
        prompt.append("\n要求：\n1. 语言要简单易懂，适合老年人理解\n2. 给出具体的行动建议\n3. 强调需要就医的情况");
        
        return callDeepSeekAPI(Arrays.asList(
            new Message("system", "你是老年健康管理专家"),
            new Message("user", prompt.toString())
        ));
    }
    
    /**
     * 构建系统提示词（注入用户上下文）
     */
    private String buildSystemPrompt(Long userId) {
        User user = userService.getById(userId);
        List<HealthRecord> recentRecords = healthRecordService.getLast7Days(userId);
        
        return String.format(
            "你是星云医疗助手的AI健康顾问。\n" +
            "当前用户：%s，年龄：%d岁\n" +
            "最近健康数据概况：%s\n" +
            "请用亲切、专业、易懂的语言回答问题。",
            user.getNickname(),
            calculateAge(user.getBirthday()),
            summarizeHealthData(recentRecords)
        );
    }
}

[5] 数据同步流程（鸿蒙特色）
    前端发送: POST /device/sync-data {deviceId, dataType, data}
    ↓
    DeviceController.syncData()
    ↓
    DeviceServiceImpl.handleSync():
    ├─ 验证设备绑定关系
    ├─ 解析同步数据（JSON格式）
    ├─ 写入对应的health_record表
    └─ 触发通知：数据已同步到其他设备
```

### 🗓️ Day 4：共病风险评估算法（6小时）⭐核心亮点

**做什么**：根据健康数据自动计算多种疾病风险等级 ⭐这是最复杂的算法

**算法规则**：

**高血压**：<120/80低风险(0分) | 120-139/80-89中风险(1分) | ≥140/90高风险(2分) | >65岁+1分

**糖尿病**：空腹血糖<6.1低(0) | 6.1-7.0中(1) | >7.0或有病史高(2) | BMI>25+1分

**跌倒**：<65岁低 | 65-75中 | >75或有跌倒史高 | 用辅助器具+1分

**衰弱/肌少症**：根据年龄、体重变化、BMI综合评分

**综合等级**：总分0-3→低(绿) | 4-7→中(黄) | 8-10→高(红)

**具体任务**：
1. RiskAssess实体类（45分钟）
2. 设计算法（2.5小时）⭐重点难点
3. RiskAssessService（1.5小时）
4. RiskAssessController（1小时）
5. 测试验证（45分钟）：准备3组测试数据

**最终效果**：
✅ 自动计算5种疾病风险，颜色直观展示
✅ 生成详细报告和建议
✅ 查看历史趋势

**验收标准**：
- [ ] 输入健康数据，返回5项风险等级
- [ ] 年轻人低风险，老年慢性病患者高风险
- [ ] 报告含数值、等级、建议

## ⚠️ 六、风险预案

**风险1：DeepSeek API失败或额度不足**（概率30%）
- 准备本地规则引擎降级方案
- 切换到预设回答模板
- 提前申请免费额度或备用Key

**风险2：前后端数据格式不一致**（概率60%）⚠️ 高概率
- Day6完成后立即整理API文档给前端B
- 约定统一规范（日期格式、命名风格）
- 使用Swagger/YApi自动生成文档
- 联调前对齐关键接口结构

**风险3：某个模块超时**（概率40%）
- 每晚评估进度，及时发现延期
- 优先保证P0任务
- P2任务可简化或砍掉
- 周末适当补进度

**风险4：数据库性能瓶颈**（概率20%）
- Day14性能测试发现
- 添加索引、大表分页、热点缓存

**风险5：团队协作问题**（概率35%）
- 每日站会10分钟
- 及时反馈阻塞
- 代码每日合并
- 分歧以产品经理D意见为准
